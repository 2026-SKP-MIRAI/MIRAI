# engine/

## 목적
전원 공동 설계·기여하는 공통 기술 레이어. TDD 적용.
서비스들이 공유하는 파서·LLM·프롬프트 인프라를 제공.

## 실행

Docker 컨테이너로 실행. 포트 8000, 진입점 `app.main:app`.

```bash
# 빌드
docker build -t mirai-engine .

# 구동 (자동 재시작)
docker run -d --restart unless-stopped -p 8000:8000 --env-file .env mirai-engine
```

상세 운영 가이드 → `engine/README.md`

## 구조
```
engine/
├── app/
│   ├── main.py       FastAPI 앱 진입점
│   ├── config.py     환경변수 (OPENROUTER_API_KEY 등)
│   ├── schemas.py    Pydantic 모델 (요청·응답 타입)
│   ├── parsers/      PDF → 텍스트 추출 (PyMuPDF)
│   ├── services/     LLM API 호출 (OpenRouter — google/gemini-2.5-flash)
│   ├── prompts/      프롬프트 템플릿 + 버전 관리
│   └── routers/      API 엔드포인트 (/resume, /interview 등)
├── tests/            pytest 테스트
│   └── fixtures/     테스트 데이터 (샘플 PDF, 예상 응답 JSON)
├── docs/             엔진 설계 문서
├── Dockerfile        컨테이너 빌드 (python:3.12-slim, non-root)
├── .dockerignore     빌드 컨텍스트 제외 목록
├── README.md         빌드·구동·환경변수 운영 가이드
└── pyproject.toml    Python 프로젝트 설정
```

## 역할
- 엔진 불변식 4개를 실제로 구현하는 레이어
- 기여는 자율 — 특정 인원 지정 없음
- 서비스 skeleton이 엔진의 API 계약을 먼저 정의 (outside-in)
- 모든 구현은 테스트 먼저: Red → Green → Refactor

## 엔진 불변식
```
1. LLM API 호출은 반드시 engine/services/ 에서만
2. PDF 파싱은 반드시 engine/parsers/ 에서만
3. 각 service는 engine을 호출만 — 내부 직접 접근 금지
4. 테스트 없는 PR 머지 금지
```

## 파서 edge case 동작 계약

| 조건 | 예외 | HTTP | 비고 |
|------|------|------|------|
| 텍스트 0자 (빈 PDF) | `EmptyPDFError` (`ParseError` 하위) | **422** | 클라이언트 파일 문제, 재업로드 유도 |
| 텍스트 레이어 없는 이미지 PDF | `ImageOnlyPDFError` (`ParseError` 하위) | **422** | 클라이언트 파일 문제, 재업로드 유도 |
| 손상·암호화 PDF | `ParseError` | **400** | 클라이언트 파일 문제, 재업로드 유도 |
| 파일 크기 > 5MB | `FileSizeError` (`ParseError` 하위) | **400** | — |
| 페이지 수 > 10 | `PageLimitError` (`ParseError` 하위) | **400** | — |
| LLM API 오류 (OpenRouter) | `LLMError` | **500** | 서버 측 오류, 재시도 유도 |
| 예기치 않은 파싱 오류 | 미분류 예외 | **500** | 라이브러리 내부 오류 등 |

예외 계층:
```
ParseError (base)
├── EmptyPDFError           → 422
├── ImageOnlyPDFError       → 422
├── FileSizeError           → 400
└── PageLimitError          → 400
LLMError (독립)             → 500
├── ReportParseError        → 500 (LLMError 상속)
├── PracticeParseError      → 500 (LLMError 상속)  ← Phase 3 추가
└── ResumeFeedbackParseError → 500 (LLMError 상속) ← Phase 3 추가
InsufficientAnswersError    → 422
```

> **TODO:** 예외 계층 세분화 — FileSizeError·PageLimitError는 파싱 전 검증 에러이므로 `ValidationError`로 분리 검토 (Beta 진입 시점에 결정)

- 파서(`app/parsers/`)는 예외를 던진다. 빈 문자열 반환 안 함.
- HTTP 변환은 `app/main.py` 전역 exception handler에서 처리.
- 서비스 레이어(`app/services/`)는 빈 문자열 입력 시 `LLMError` throw (파서가 걸러주므로 실제 도달하지 않아야 함).

## API 엔드포인트 계약

### `GET /` (헬스체크)
- ALB 타겟 그룹 헬스체크용
- 응답: `{ status: "ok" }`

### `/api/resume/questions` (POST, multipart/form-data)
- 입력: PDF 파일 (`file`)
- 출력: `{ questions: QuestionItem[], meta: Meta }`
- 에러: 400 (파일 없음·비PDF·크기초과·페이지초과), 422 (빈PDF·이미지PDF), 500 (LLM오류)

### `/api/interview/start` (POST, JSON)
- 입력: `{ resumeText: str, personas: ["hr"|"tech_lead"|"executive"], mode: "panel" }`
- 출력: `{ firstQuestion: QuestionWithPersona, questionsQueue: QueueItem[] }`
- 에러: 400 (필드 누락·빈 문자열), 500 (LLM오류)

### `/api/interview/answer` (POST, JSON)
- 입력: `{ resumeText, history: HistoryItem[], questionsQueue: QueueItem[], currentAnswer }`
- 출력: `{ nextQuestion: QuestionWithPersona | null, updatedQueue: QueueItem[], sessionComplete: bool }`
- 에러: 400 (필드 누락), 500 (LLM오류)
- 주의: `questionsQueue`가 비면 LLM 호출 없이 `sessionComplete=True` 즉시 반환

### `/api/interview/followup` (POST, JSON)
- 입력: `{ question, answer, persona: "hr"|"tech_lead"|"executive", resumeText }`
- 출력: `{ followupType: "CLARIFY"|"CHALLENGE"|"EXPLORE", followupQuestion, reasoning }`
- 에러: 400 (필드 누락), 500 (LLM오류)

### `/api/practice/feedback` (POST, JSON) — Phase 3
- 입력: `{ question: str, answer: str, previousAnswer?: str }`
- 출력: `{ score: int(0-100), feedback: {good: str[], improve: str[]}, keywords: str[], improvedAnswerGuide: str, comparisonDelta?: { scoreDelta: int, improvements: str[] } }`
- 에러: 400 (필드 누락/빈 문자열), 500 (LLM 오류/JSON 파싱 실패)
- timeout: 30s, max_tokens: 2048
- previousAnswer 없으면 comparisonDelta=null, 있으면 comparisonDelta 포함
- 프롬프트: `engine/app/prompts/practice_feedback_v1.md` (단일), `practice_feedback_retry_v1.md` (재답변)

### `/api/resume/feedback` (POST, JSON) — Phase 3
- 입력: `{ resumeText: str, targetRole: str }`
- 출력: `{ scores: { specificity, achievementClarity, logicStructure, roleAlignment, differentiation }, strengths: str[2-3], weaknesses: str[2-3], suggestions: [{ section, issue, suggestion }] }`
- 에러: 400 (필드 누락/빈 문자열), 500 (LLM 오류·JSON 파싱 실패)
- timeout: 30s, max_tokens: 2048
- 프롬프트: `engine/app/prompts/resume_feedback_v1.md`
- scores 엄격 검증: 5개 키 중 하나라도 누락 또는 null → `ResumeFeedbackParseError` (500). silent fallback 없음
- scores 범위 검증: 0~100 초과 시 `ResumeFeedbackParseError` (500). clamp 보정 없음
- suggestions 최소 1개 보장: LLM 빈 배열 반환 시 fallback SuggestionItem 자동 삽입

### `/api/report/generate` (POST, JSON) — Phase 2
- 입력: `{ resumeText: str, history: HistoryItem[] }` (history 최소 5개)
- 출력: `{ scores: AxisScores, totalScore: int, summary: str, axisFeedbacks: AxisFeedback[8], growthCurve: null }`
- 에러: 400 (필드 누락), 422 (history < 5개 → InsufficientAnswersError), 500 (LLM오류·파싱실패)
- 전략: 싱글 LLM 1회 호출 (비용 절감, 축 간 일관성)
- 8축: communication, problemSolving, logicalThinking, jobExpertise, cultureFit, leadership, creativity, sincerity
- type 보정: score >= 75 → "strength", score < 75 → "improvement" (LLM 오분류 자동 교정)
- growthCurve: Phase 3 placeholder (항상 null)
- 프롬프트: `engine/app/prompts/report_evaluation_v1.md`
- **⚠️ 서비스 측 타임아웃 주의**: 엔진 내부 LLM timeout=60s, 예상 응답시간 12–18s.
  서비스(Next.js)에서 이 엔드포인트 호출 시 fetch timeout을 **90s 이상**으로 설정해야 함.
  기본값(30s)으로 호출하면 LLM 응답 전에 서비스 측이 먼저 타임아웃 발생.
