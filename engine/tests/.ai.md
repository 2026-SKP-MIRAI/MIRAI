# engine/tests/

## 목적
엔진 전체 pytest 테스트. TDD 원칙 — 구현 전 테스트 먼저 작성.

## 구조
```
tests/
├── unit/
│   ├── parsers/          PDF 파서 단위 테스트
│   └── services/
│       ├── test_interview_service.py   면접 서비스 단위 테스트 (16개)
│       ├── test_report_service.py      리포트 서비스 단위 테스트 (17개) ← Phase 2
│       ├── test_practice_service.py    연습 피드백 서비스 단위 테스트 (13개) ← Phase 3
│       ├── test_feedback_service.py    서류 피드백 서비스 단위 테스트 (12개) ← Phase 3
│       ├── test_llm_service.py         LLM 서비스 단위 테스트
│       └── test_output_parser.py       출력 파서 단위 테스트
├── integration/
│   ├── test_resume_questions_route.py  자소서 질문 생성 통합 테스트
│   ├── test_interview_router.py        면접 라우터 통합 테스트 (11개)
│   ├── test_report_router.py           리포트 라우터 통합 테스트 (8개) ← Phase 2
│   ├── test_practice_router.py         연습 피드백 라우터 통합 테스트 (5개) ← Phase 3

└── fixtures/
    ├── input/            샘플 PDF 등 입력 픽스처 (gitignored)
    └── output/
        ├── mock_report_response.json              리포트 LLM 응답 픽스처 ← Phase 2
        ├── mock_history_5items.json               면접 히스토리 5개 픽스처 ← Phase 2
        ├── mock_practice_feedback_single.json     연습 피드백 단일 응답 픽스처 ← Phase 3
        ├── mock_practice_feedback_retry.json      연습 피드백 재답변 응답 픽스처 ← Phase 3
        └── mock_practice_feedback_overlong.json   슬라이싱 테스트용 픽스처 ← Phase 3
```

## 역할
- Red → Green → Refactor 사이클 준수
- 테스트 없는 PR 머지 금지 (CI 차단)
- `fixtures/input/`에 실제 PDF 커밋 금지 (.gitignore 적용) — 합성 픽스처만 허용
- `fixtures/output/`의 JSON 픽스처는 LLM 모킹용 — 커밋 가능

## 모킹 전략
- LLM 호출 모킹: `patch("app.services.llm_client.OpenAI", return_value=make_mock_llm(content))`
- 모든 서비스가 `llm_client.py`를 통해 LLM을 호출하므로 단일 mock 경로 사용

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
