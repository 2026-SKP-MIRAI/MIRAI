# engine/tests/unit/parsers/

## 목적
PDF 파서 단위 테스트. TDD 원칙 — 구현 전 테스트 먼저 작성.

## 구조
```
unit/parsers/
└── test_pdf_parser.py    PDF 파서 단위 테스트 14개 (OCR fallback 포함)
```

## 테스트 목록 (14개)

### 기존 파서 검증 (9개)
- `test_parse_pdf_success` — 텍스트 PDF 정상 파싱
- `test_empty_pdf_raises_empty_pdf_error` — 빈 PDF → EmptyPDFError
- `test_corrupted_pdf_raises_parse_error` — 손상 파일 → ParseError
- `test_large_file_raises_file_size_error` — 5MB 초과 → FileSizeError
- `test_too_many_pages_raises_page_limit_error` — 11페이지 → PageLimitError
- `test_exactly_5mb_passes` — 5MB 이하 통과
- `test_5mb_plus_1byte_fails` — 5MB+1바이트 → FileSizeError
- `test_exactly_10_pages_passes` — 10페이지 통과
- `test_11_pages_fails` — 11페이지 → PageLimitError

### OCR fallback 테스트 (5개)
- `test_ocr_extracts_text_from_image_pdf` — 이미지 PDF → OCR → ParsedResume (`@requires_tesseract`)
- `test_ocr_empty_result_raises_image_only_error` — 판독 불가 이미지 → ImageOnlyPDFError (`@requires_tesseract`)
- `test_ocr_not_run_when_text_exists` — 텍스트 레이어 있으면 OCR 미호출 (mock)
- `test_ocr_internal_error_raises_image_only_error` — `_ocr_fallback` 빈 반환 → ImageOnlyPDFError
- `test_ocr_fallback_returns_empty_on_exception` — `_ocr_fallback` 내부 예외 → 빈 문자열

## 픽스처
- `minimal_pdf_bytes` — 텍스트 포함 PDF (conftest.py)
- `ocr_target_pdf_bytes` — 800x200 이미지, 36pt 폰트, "Hello OCR Test Resume" (conftest.py)
- `unreadable_image_pdf_bytes` — 1x1 흰색 픽셀 이미지 PDF (conftest.py)

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
