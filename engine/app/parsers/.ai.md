# engine/app/parsers/

## 목적
PDF 파싱 레이어. 자소서 PDF에서 텍스트·구조를 추출하는 유일한 경계.

## 구조
```
parsers/
└── pdf_parser/    PDF → 텍스트 추출 모듈 (Week 1 산출물)
    ├── tests/     TDD: 테스트 먼저 작성
    └── ...
```

## 역할
- 엔진 불변식 2번: PDF 파싱은 반드시 여기서만
- 입력: PDF 파일 경로 또는 바이트스트림
- 출력: 구조화된 텍스트 (섹션·단락 구분 포함)
- 이 경계 밖에서 PDF를 직접 파싱하면 CI 차단

## 작업 전 확인
- `engine/.ai.md` — 타입 계약·예외 계층 확인
- 기존 테스트 파일 확인 후 Red-Green-Refactor 사이클 준수

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
