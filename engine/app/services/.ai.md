# engine/app/services/

## 목적
LLM API 레이어. 모든 LLM 호출의 단일 진입점.

## 구조
```
services/
├── llm_client.py         공통 LLM 호출 유틸리티 ← Phase 2 추가
│                         call_llm(prompt, *, model, timeout, max_tokens, error_message)
│                         parse_object(raw, required_keys)
├── llm_service.py        자소서 질문 생성 서비스 (Phase 1)
│                         generate_questions(resume_text, *, target_role=None, model, max_input_chars, timeout_seconds) -> list[QuestionItem]
├── role_service.py       직무 추출 서비스 (#113)
│                         extract_target_role(resume_text, *, model, max_input_chars, timeout_seconds) -> str
├── interview_service.py  면접 서비스 (Phase 1) — llm_client 사용
├── report_service.py     리포트 생성 서비스 (Phase 2) — llm_client 사용
├── practice_service.py   연습 피드백 서비스 (Phase 3) — llm_client 사용
├── feedback_service.py   서류 피드백 서비스 (Phase 3) — llm_client 사용
│                         generate_resume_feedback(resume_text, target_role=None, *, model=None) -> ResumeFeedbackResponse
└── output_parser.py      LLM 응답 파서
```

## 역할
- 엔진 불변식 1번: LLM 호출은 반드시 여기서만
- 입력: 프롬프트 + 파라미터
- 출력: LLM 응답 (파싱 전 raw 또는 구조화)
- `llm_client.py`가 OpenAI 호출의 단일 진입점 — 모든 서비스가 이를 통해 호출
- 모델 전환·토큰 관리·에러 처리를 여기서 중앙 관리

## llm_client.py 인터페이스
```python
call_llm(
    prompt: str,
    *,
    model: str | None = None,       # None이면 settings.openrouter_model 사용
    timeout: float = 30.0,          # 리포트 등 긴 응답은 60.0 전달
    max_tokens: int = 2048,         # 기본 2048; 필요 시 더 높은 값 전달
    error_message: str = "처리 중 오류가 발생했습니다.",  # 맥락별 커스텀 가능
) -> str

parse_object(raw: str, required_keys: list[str] | None = None) -> dict
```

## 작업 전 확인
