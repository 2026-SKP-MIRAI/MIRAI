# engine/app/prompts/

## 목적
프롬프트 관리 및 버전 관리. 프롬프트 변경 이력을 추적 가능하게 보존.

## 구조
```
prompts/
├── interview_hr_v1.md            HR 페르소나 면접 질문 생성
├── interview_tech_lead_v1.md     기술팀장 페르소나 면접 질문 생성
├── interview_executive_v1.md     경영진 페르소나 면접 질문 생성
├── interview_followup_v1.md           꼬리질문 판단
├── report_evaluation_v1.md            8축 역량 평가 리포트 생성 (Phase 2)
├── practice_feedback_v1.md            연습 피드백 — 단일 답변 (Phase 3)
├── practice_feedback_retry_v1.md      연습 피드백 — 재답변 비교 (Phase 3)
└── resume_feedback_v1.md              서류 피드백 — 5개 항목 점수 + 강점·약점·개선 제안 (Phase 3)
```

## 버전 이력
| 파일 | 버전 | 추가일 | 설명 |
|------|------|--------|------|
| report_evaluation_v1.md | v1 | 2026-03-11 | 8축 역량 평가, 싱글 LLM 1회 호출 전략 |
| practice_feedback_v1.md | v1 | 2026-03-12 | 연습 모드 단일 답변 피드백 (score, feedback, keywords, guide) |
| practice_feedback_retry_v1.md | v1 | 2026-03-12 | 재답변 비교 피드백 (+ comparisonDelta) |
| resume_feedback_v1.md | v1 | 2026-03-13 | 서류 피드백 — 5개 항목 점수 + 강점·약점·개선 제안 |

## 역할
- 프롬프트를 코드처럼 버전 관리
- 파일명에 버전 포함: _v1, _v2 등
- llm_service가 여기서 프롬프트를 로드
- 프롬프트 변경 시 반드시 기존 버전 파일 보존 + 새 버전 추가

## 작업 전 확인
- 기존 프롬프트 버전 확인 후 새 버전 번호 결정
- 변경 이유를 파일 상단 주석에 기록

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
