# MirAI 워크플로우 가이드 (커맨드 버전)

> Claude Code 슬래시 커맨드를 사용하는 자동화된 워크플로우. 처음 기여하는 팀원을 위한 가이드.

## 처음 한 번만: 환경 설정

```bash
gh auth login
```

---

## 한 줄 요약

```
/backlog-issue → (멘토 Ready 이동) → /start-issue → 구현 (테스트 먼저) → /finish-issue → (멘토 머지) → /cleanup-issue
```

---

## 보드 컬럼

| 컬럼 | 의미 | 누가 이동시키나 |
|------|------|-----------------|
| **Backlog** | 이슈 생성 직후. 아직 시작하면 안 됨 | 자동 (이슈 생성 시) |
| **Ready** | 멘토 확인 완료. 지금 바로 착수 가능 | 멘토 |
| **In Progress** | 내가 작업 중 | 자동 (이슈 Assign 시) |
| **In Review** | PR 올린 상태. 멘토 리뷰 대기 | 직접 이동 |
| **Done** | 머지 완료 | 자동 (PR 머지 시) |
| **Drop** | 안 하기로 결정 | 멘토 또는 본인 |

> **내가 할 수 있는 이슈 = Ready 컬럼에 있는 것.**
> Backlog는 멘토가 Ready로 옮겨줄 때까지 기다린다.

---

## 1단계: 이슈 생성

```
/backlog-issue
```

- 인수 없이 실행하면 Claude가 type, 제목, 배경을 순서대로 질문한다
- 완료 기준 초안을 제안하고 확인 후 이슈를 생성한다
- 생성된 이슈는 자동으로 Backlog 컬럼에 추가된다

---

## 2단계: 내 이슈 확인

```bash
gh issue list --assignee @me
```

멘토가 Ready로 이동시킨 이슈를 확인한다.

---

## 3단계: 작업 시작

```
/start-issue <이슈번호|짧은이름>
```

이슈번호만 넣으면 Claude가 제목에서 브랜치 이름을 자동으로 만든다:

```
/start-issue 15
```

내부에서 자동으로 처리:
- 이슈 제목 조회 → type·짧은이름 자동 추출
- `.worktree/000015-pdf-upload/` 생성 (워크트리)
- `feat/000015-pdf-upload` 브랜치 생성
- `docs/work/active/000015-pdf-upload/` 생성 (작업 폴더)
- 이슈를 자신에게 Assign (→ In Progress 자동 이동)

완료 후 안내하는 경로로 이동:
```bash
cd .worktree/000015-pdf-upload
```

---

## 4단계: 구현 (테스트 먼저)

```
이슈의 AC(인수 조건)를 먼저 읽는다
  ↓
AC를 테스트 코드로 작성 (Red)
  ↓
테스트 통과하는 코드 작성 (Green)
  ↓
리팩터링 (Refactor)
  ↓
해당 디렉토리 .ai.md 업데이트
```

---

## 5단계: PR 생성

```
/finish-issue
```

워크트리 안에서 인수 없이 실행하면 현재 브랜치를 자동으로 감지한다.
이슈번호나 짧은이름을 직접 넣어도 된다: `/finish-issue 15`

내부에서 자동으로 처리:
- 변경사항을 분석해 커밋 메시지 자동 생성 (확인 후 진행)
- 커밋 + push
- PR 생성 (본문에 `Closes #15` 자동 포함)

---

## 6단계: 리뷰 기다리기

- Reviewer로 멘토가 자동 지정됨
- 멘토가 코멘트 남기면 수정 후 다시 push
- 추가 커밋은 같은 브랜치에 push하면 PR에 자동 반영

---

## 7단계: 머지

멘토 승인 후 멘토가 Squash merge.
머지되면 브랜치 자동 삭제, 이슈 자동 Close, Projects 보드 Done 이동.

---

## 8단계: 정리

```
/cleanup-issue
```

인수 없이 실행하면 머지 완료된 워크트리를 전부 자동 감지해 일괄 정리한다.
특정 이슈만 정리하려면: `/cleanup-issue 15`

내부에서 자동으로 처리:
- 전체 워크트리 스캔 → CLOSED 이슈 감지
- 정리 대상 목록 확인 후 워크트리 + 로컬 브랜치 삭제

---

## 막혔을 때

1. 이슈에 코멘트로 질문 남기기 (멘토가 비동기로 확인)
2. PR을 Draft로 열고 `[WIP]` 표시
3. 진행 불가 시 이슈에 `blocked` 라벨 추가
