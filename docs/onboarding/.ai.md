# docs/onboarding/

## 목적
새 팀원 온보딩 가이드. 개발 환경 설정·기여 프로세스·컨벤션 설명.

## 구조
```
onboarding/
├── workflow.md                  수동 git 명령어 기반 기여 워크플로우
├── workflow(command ver).md     슬래시 커맨드 기반 자동화 워크플로우
├── commands.md                  슬래시 커맨드 레퍼런스 (인수·예시 상세)
└── (추가 가이드 파일들 — 작성 예정)
    예: setup.md        로컬 개발환경 세팅
        contributing.md PR·커밋·브랜치 전략
        conventions.md  코딩 컨벤션
```

## 역할
- 프로젝트 합류 시 가장 먼저 읽는 곳
- docker-compose 실행 방법 포함 예정 (Week 1 산출물)
- 에이전트도 여기를 기준으로 개발 환경 파악

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
