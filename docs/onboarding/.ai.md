# docs/onboarding/

## 목적
새 팀원 온보딩 가이드. 개발 환경 설정·기여 프로세스·컨벤션 설명.

## 구조
```
onboarding/
└── (환경 설정 및 기여 가이드 파일들 — 작성 예정)
    예: setup.md        로컬 개발환경 세팅
        contributing.md PR·커밋·브랜치 전략
        conventions.md  코딩 컨벤션
```

## 역할
- 프로젝트 합류 시 가장 먼저 읽는 곳
- docker-compose 실행 방법 포함 예정 (Week 1 산출물)
- 에이전트도 여기를 기준으로 개발 환경 파악
