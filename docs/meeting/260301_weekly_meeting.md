- AI 환각 검증이 중요
- AI 생성 콘텐츠 표시 규제가 생길 가능성 있음
- 출력 결과를 맹신하지 말고 검증하는 습관 필요

## 5. 기획서 강화 + TODO

기획서 품질이 AI 활용 결과를 결정한다. TODO를 명확하게 정의하고 추적한다.

가능한 한 GitHub로 관리한다. Notion을 쓰는 데는 이유가 있어야 한다 (비개발자 협업, 공유 문서 등).

기획부터 출시까지 전 과정을 다루는 역할. 고객 에이전트와 QA 에이전트를 직접 만들어 자체 평가한다.

- 풀스택 빌더는 아직 정의된 직군이 아님 — 새로운 포지션

## 10. 채용시장

- 신입 공고가 극단적으로 적음
- Forward Deployed Engineer 개념 참고

## 11. 프로젝트 목표

- 한 달 안에 완성
- 사용자 10명 확보
- 모객 → 사용 → 피드백 라이프사이클 완성

## 12. SSoT + 하네스

- 레포지토리 = 지식의 Single Source of Truth
- 아키텍처를 기계적으로 강제하는 환경 설계 필요
- 규칙을 문서로만 쓰지 말고 자동화로 강제

## 13. 5가지 역량

1. **환경설계** — AI가 잘 동작하는 환경을 만드는 능력
2. **의도명시** — 무엇을 원하는지 정확히 표현
3. **피드백루프** — 빠른 검증과 반복
4. **아키텍처판단** — 구조적 결정을 내리는 능력
5. **인간판단 복리화** — 올바른 판단이 쌓여 복리로 가치 증가

## 14. 운영방식/금지사항

- AI 실수 = 환경설계 실패로 봐야 한다
- 멘토 역할: 방향 제시, 코드 리뷰 아님
- CHACHA95 (언급된 키워드, 구체적 내용 추후 확인 필요)
