# 2/21 멘토님 온라인 미팅 정리

## 1. 운영/환경 세팅

원격 개발환경 터널링 접속 준비. 외부에서도 개발 환경에 접근할 수 있도록 구성.

## 2. Claude 세팅

- GitHub my Claude setup
- Oh My Claude 설치
- 통계로 증명하는 방식으로 사용 효과 확인

## 3. 워크플로 원칙

```
Plan → 실행 → 검토 → 실패 시 Plan 복귀
```

- 복잡한 작업 = Plan mode 진입
- 멀티에이전트 활용
- 실패했을 때 코드부터 고치지 말고 Plan으로 돌아간다

## 4. 원스텝을 위한 명세

- Figma 같은 사람용 UI SaaS가 병목이 됨
- MCP로 AI가 툴을 직접 연결 가능
- 명세가 충분하면 AI가 원스텝으로 처리

## 5. 이케아 효과

직접 짠 코드에 애착이 생겨 AI 활용을 방해한다. AI로 개발하면 결국 남는 것은 기획과 명세다. 코드 애착을 버려야 한다.

## 6. 꿀팁

- "링크 내용 중 우리가 하고 있는 것 / 안 하는 것"을 그대로 Claude Code에 입력하면 효과적
- shell alias 우선순위 설정

## 7. 프로젝트 피드백

- 거시지표(청년실업률 등)를 서비스 목표로 쓰는 건 약점 — 단일 제품이 거시지표를 직접 움직이기 어려움
- 제너럴리스트 선호 — 전문화보다 전체 제품 라이프사이클 이해
- 기술보다 프로덕트 목적이 중요
- FastAPI/Pydantic을 왜 쓰는지 설명할 수 있어야 함

## 8. 일론 머스크 5단계

1. 요구사항 의심하기
2. 불필요한 부분 줄이기
3. 단순화하기
4. (이후 단계)
5. (이후 단계)

1~3단계가 핵심. MVP 정의 예시: **"면접 질문 1개 + 점수 평가"**

→ 극단적으로 범위를 줄이는 것이 올바른 MVP 접근이다.
