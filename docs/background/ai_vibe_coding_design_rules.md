# AI/바이브 코딩 디자인 6가지 규칙

## 핵심 요약

1. AI 코딩 결과물의 디자인을 개선하려면 **컬러 팔레트 선택, Stitches 활용, MCP 대신 Skills 사용, Shadcn UI 사용** 등 6가지 핵심 규칙을 따라야 한다.
2. 색상 선택 시 **coolors.co에서 제공하는 조화로운 팔레트를 사용**하고, 이 중 메인·서브 컬러 두 가지만 활용하며, 헥스 코드를 정확히 입력하는 것이 중요하다.
3. AI 코딩 시 디자인 일관성을 유지하려면 **MCP 대신 Skills를 통해 디자인 가이드라인을 문서화**하여 적용하고, 컴포넌트 라이브러리로는 AI와 궁합이 좋은 **Shadcn UI**를 사용하는 것이 좋다.

## 왜 AI 코딩 결과물은 디자인이 구려 보일까?

색깔이 너무 많고, 버튼 크기나 여백이 제각각인 경우가 많다. 디자인 감각이 아닌 **지키기만 하면 되는 규칙**이 필요하다.

## 규칙 1: 색상은 직접 고르지 마라

AI는 아직 색 조합을 잘 맞추지 못한다. **[coolors.co](https://coolors.co)** 에서 스페이스바를 누르면 자동으로 조화로운 팔레트가 생성된다.

- 팔레트에서 **메인 컬러 1개 + 서브 컬러 1개**, 총 2가지만 사용
- 나머지는 흰색·검정·회색 계열로 채움
- 헥스 코드를 프롬프트에 **그대로** 넣을 것 (직접 수정 금지)
- 메인 컬러 → CTA 버튼·강조 / 서브 컬러 → 배경·보조 요소

## 규칙 2: UI 레이아웃은 Stitches 활용

AI UI 생성 도구 **Stitches**로 프롬프트 한 줄에 웹사이트·앱 UI 화면 생성.

- coolors에서 받은 헥스 코드를 **정확하게** 넣을 것
- 레퍼런스 사이트 묘사는 "토스처럼"이 아닌 **영문으로 구체적으로** 묘사

## 규칙 3: MCP 대신 Skills로 디자인 가이드라인 문서화

Claude Code에서 디자인 시 MCP 사용은 비추천.
- MCP: 토큰 소진 → 디자인 일관성 저하, 수정할수록 버튼 스타일·간격·컬러 무너짐
- **Skills**: 컬러·여백·폰트 사이즈 등 디자인 규칙을 문서로 미리 정의 → Claude Code가 코드 생성 시 참고, 일관된 디자인 유지, 토큰 소모 적음

## 규칙 4: Next.js 프로젝트엔 MUI 대신 Shadcn UI

| | MUI | Shadcn UI |
|--|-----|-----------|
| 설치 방식 | npm 패키지 | 소스 코드 직접 복사 |
| AI 수정 | 어려움 (복잡한 override) | 쉬움 (코드 직접 수정) |
| 커스터마이징 | 복잡 | 간단 (config 파일에 컬러 지정) |
| 바이브 코딩 궁합 | 낮음 | 높음 |

## 성공적인 AI 코딩 디자인 순서

1. coolors.co에서 팔레트 선택 → 메인/서브 헥스 코드 추출
2. Next.js 프로젝트 생성 시 Shadcn UI 세팅 + tailwind config에 컬러 추가
3. 화면 레이아웃 필요 시 Stitches 사용 (헥스 코드 + 영문 레퍼런스 묘사)
4. Claude Code 바이브 코딩 시 MCP 대신 Skills에 디자인 가이드라인 작성

> **기능 먼저, 디자인 나중은 금물.** 처음부터 컬러와 Shadcn UI를 세팅하고 시작할 것.
