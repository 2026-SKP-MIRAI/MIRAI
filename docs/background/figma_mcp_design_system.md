# 피그마 MCP + 디자인 시스템으로 AI 코딩 완성도 높이기

## 핵심 요약

1. 피그마 MCP와 데브 모드를 활용하여 디자인 시스템을 구축하면, 디자인을 효율적으로 관리하고 재사용 가능한 컴포넌트를 제작하여 **하드코딩의 비효율성을 줄일 수 있다**.
2. AI/바이브 코딩 시 깨끗한 코드를 얻고 할루시네이션을 줄이기 위해서는, **디자인 시스템을 기반으로 토큰 및 컴포넌트를 구성**하고 이를 활용하여 코딩하는 것이 중요하다.
3. 스크린 플로우는 원하는 웹사이트를 해상도별로 캡처하고 피그마로 편집 가능한 디자인 형태로 가져와 **디자인 레퍼런스 작업의 효율성을 극대화**하는 서비스다.

## 왜 디자인 시스템인가

MVP 수준에서는 하드코딩이 괜찮을 수 있지만, 제품 관점에서 컴포넌트 단위로 관리해야 효율이 좋고 고도화에도 유리하다. 하드코딩 시 발생하는 유지보수 비용과 AI 코딩 시 토큰·시간 소모를 줄일 수 있어, 서비스 제작 시 디자인 시스템 구축은 필수다.

- 바이브 코딩 시 디자인 시스템을 활용하는 것과 그렇지 않은 것의 결과물 차이는 매우 큼
- 디자인 레이어명이나 컴포넌트가 잘 정리되어 있을수록 AI 코딩 효율이 높아짐

## 피그마 MCP 연결 및 토큰 구성

- 피그마 MCP 사이트에서 코드를 복사하여 커서 채팅창에 붙여 넣거나, 설정 탭에서 파일 추가로 활용
- MCP 연결 시 스타일 가이드의 노드 정보와 메타데이터를 읽어와 디자인 시스템 구성 가능
- **데브 모드 활성화** → 스타일 가이드를 섹션 단위로 묶어 한 번에 많은 스타일 정보 읽기
- 스페이스, 컬러, 텍스트 스타일을 **토큰으로 구성**하면 확장이 쉽고 재사용성이 높아짐

## 컴포넌트 구성 및 외부 소스 활용

- 프로퍼티가 적용된 컴포넌트 세트를 활용해 최소한의 컴포넌트로 전체 관리
- 디자인 시스템을 직접 구축하기 어렵다면 커뮤니티 무료 소스 적극 활용
  - Shadcn 키트, 글로벌 기업 배포 소스
  - `designsystems.com` — 유명 디자인 시스템 원본을 피그마로 바로 복사 가능

## 피그마 데브 모드를 활용한 React 코드 구현

- 데브 모드에서 컴포넌트 섹션 선택 → 구성한 변수를 토대로 React 코드 구현 요청
- 커서가 스크린샷을 제시해 선택한 화면 확인 등 업데이트된 기능 제공
- 초기 단계부터 레이어명·컴포넌트 체계 정리가 불필요한 토큰 소모를 줄임

## 디자이너-개발자 협업의 변화

- 과거: UI 디자인 후 어노테이션 작성 → 개발자에게 요청
- 현재: 간단한 논의 후 **디자이너가 직접 시스템 파일 수정 → 깃에 올림**
- 개발자는 서버 안정성·인프라 등 핵심 업무에 집중 가능

## 스크린 플로우 (Screen Flow)

원하는 웹사이트를 해상도별로 한 번에 캡처하고, 피그마에서 편집 가능한 디자인으로 가져올 수 있는 서비스.

- 웹사이트 링크 입력 → 다양한 depth의 페이지 캡처 + 아카이브
- 피그마 플러그인으로 HTML 원본을 가져와 수정 가능 (팝업 등 복잡한 요소 포함)
- 레퍼런스 수집 및 아이디어 구체화 과정에서 반복 캡처·정리 작업 대폭 감소
- 미국 디자이너 인터뷰에서 "작업 시간 50% 단축" 피드백

## AI 입문자를 위한 조언

- 거창한 사례에 압도되지 말고, **자신이 겪는 사소한 문제를 제품으로 해결하는 것부터** 시작
- 피그마 플러그인 같이 쉽게 접근할 수 있는 도구부터 활용
- 모든 기술을 다 알 필요는 없음 — 내 일에 어떻게 접목할지 고민하는 과정이 중요
