# 데이터 구조가 알고리즘을 결정한다 — 롭 파이크의 다섯 번째 규칙

## 핵심 요약

1. 견고하고 확장 가능한 코드를 위한 핵심은 **올바른 데이터 구조 설계**에 있다.
2. 데이터 구조가 취약하면 if문이 늘어나 로직이 복잡해지며, 이는 **아키텍처의 취약성을 막는 가장 중요한 방어선**이다.
3. 올바른 데이터 구조 적용은 취약한 접근을 피하고, 단순하지만 효과적인 구조를 사용하며, **잘못된 상태를 막아 코드 안정성과 확장성을 높인다**.

## 배경: 롭 파이크의 다섯 번째 규칙

벨 연구소에서 유닉스 개발에 기여하고 Go 언어를 공동 설계한 롭 파이크는 다섯 가지 프로그래밍 규칙을 정리했다. 그 중 핵심은 다섯 번째 규칙:

> **"데이터 구조를 제대로 고르면 알고리즘은 자연스럽게 풀린다."**

우리는 이전보다 훨씬 강력한 연산 능력과 고수준 추상화를 손에 넣었지만, 정작 제대로 작동하는 로직을 만드는 데에는 그에 걸맞은 시간을 들이지 못하고 있다. 프로그래밍 선구자들은 극심한 메모리 제약 속에서도 우리가 점점 잃어가는 사고의 명료함을 가지고 있었다.

## 올바른 데이터 구조 설계 원칙

### 1. 취약한 접근을 피하라

단순한 배열에서 원시 값을 뒤져 원하는 대상을 찾는 방식은, 새로운 변수나 역할을 추가해야 하는 순간 쉽게 한계에 부딪혀 코드가 흔들리기 시작한다.

### 2. 단순하지만 효과적인 구조를 사용하라

데이터 구조는 거창해야만 효과적인 것이 아니다. Enum처럼 단순한 구조로 역할을 정의하고 클래스 안에 담아두기만 해도 다뤄야 할 범위를 훨씬 명확하게 좁힐 수 있다.

### 3. 잘못된 상태를 막아라

처음부터 데이터를 정확하게 정의해두면 사용자나 역할 같은 정보가 잘못된 상태에 빠질 가능성을 크게 줄일 수 있다. 새로운 기능을 추가하더라도 기존 코드를 흔들지 않고 안정적으로 확장 가능하다.

## 결론

우리에게 부족한 것은 도구가 아니다. 오히려 우리는 도구에 파묻혀 있다. 도구를 제대로 다루기 위해 필요한 **기본적인 사고 방식**을 다시 돌아봐야 한다.
