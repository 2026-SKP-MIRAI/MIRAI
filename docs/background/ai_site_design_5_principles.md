## 한 줄 요약

"홈페이지 만들어줘"로는 절대 안 나오는 퀄리티 — AI 사이트 100개 이상을 납품하며 찾아낸, 같은 AI로 전혀 다른 결과를 뽑는 원칙 5가지.

## 핵심 인사이트

AI가 못 만드는 게 아니라, **시키는 방법**을 모르는 것이다.
도구는 계속 변하지만, **원칙을 아는 사람만 더 좋은 결과를 뽑아낸다.**

---

## 원칙 1 — 코딩부터 하지 않는다

코딩 전에 **3가지 조사**를 먼저 AI에게 시킨다.

### 1-1. 대상 조사
- 사이트의 주인공(클라이언트 or 나)을 AI가 깊이 파악하게 한다
- 예: 블로그 283개 → AI가 보고서 작성 → 해당 내용이 그대로 홈페이지 문구로 사용됨
- 프롬프트 예시: `"[업체명] 도배 업체 리서치 해줘. 네이버 블로그·인스타·포트폴리오 전부 조사. 주력 분야, 강점, 고객 후기, 경쟁사 대비 차별점 정리."`

### 1-2. 도메인 조사
- 해당 업종 사이트들의 패턴과 목적을 파악
- 예: 도배 사이트 → "로그인·결제 불필요, 전화 한 통으로 끝" → **목적이 '팔리는 사이트'가 아니라 '전화 오게 만드는 사이트'**
- 이 한 줄을 아느냐 모르냐가 홈페이지 구조 전체를 바꾼다
- 어워드 사이트(Awwwards 등) 카테고리 분석 포함

### 1-3. 컴포넌트 조사
- 버튼·카드·갤러리 같은 UI 부품 라이브러리 매핑
- 대표 라이브러리: shadcn/ui, Magic UI 등
- 프롬프트 예시: `"조사 결과 바탕으로 shadcn/ui·Magic UI에서 구현 가능한 컴포넌트를 섹션별(히어로·갤러리·포트폴리오)로 매핑해줘."`
- 소요 시간: 30분~1시간 (생략하면 공장식 디자인이 나옴)

---

## 원칙 2 — 내 눈으로 직접 고른다

AI가 레퍼런스를 찾아왔다면, 이번엔 **직접 눈으로 보고 고른다.**

### 21st.dev 컴포넌트 쇼룸
- 여러 컴포넌트 라이브러리를 한 곳에 모아 놓은 쇼룸
- 직접 보는 것이 AI 추천보다 빠르고, **내 감각이 결과물에 들어감**

### 아무 사이트든 DOM 뜯어서 가져오기
1. 마음에 드는 사이트 → 개발자 도구 열기
2. 화살표 아이콘으로 원하는 영역 클릭 → HTML 구조 복사 (`Copy Element`)
3. AI에게 붙여넣기 → AI가 내 코드로 재구현
- **"디자인의 뼈대만 가져오고, 살은 AI가 붙인다"**
- 인터넷에 있는 모든 사이트가 레퍼런스가 된다

## 원칙 3 — 히어로 5개 만들기 (2라운드)

전체 페이지를 만들지 않는다. **히어로 섹션 하나**로 먼저 전체 방향을 정한다.

### 1라운드 — 테마 정하기
- 최고 모델 + Thinking 모드로 히어로 5가지 테마 생성
- 상단 토글 버튼으로 전환하며 비교
- 원칙 1·2의 레퍼런스·소재를 컨텍스트로 함께 제공
- 마음에 드는 테마가 나올 때까지 반복 생성
- **가장 많은 토큰과 시간을 이 단계에 투자**

### 2라운드 — 색상 정하기
- 확정된 테마 안에서 컬러 조합 5가지 생성
- 같은 레이아웃·구조, 컬러만 다르게 → 빠른 비교 가능
- 외주 시: 토글 링크를 클라이언트에게 공유 → 직접 선택하게 함

> **핵심**: 한 번에 모든 것을 정하려 하면 결정을 못 한다.
> 1라운드에서 큰 방향, 2라운드에서 디테일을 확정한다.

## 원칙 4 — 미디어와 캔버스

코드만으로는 임팩트에 한계가 있다. **미디어(영상)** 또는 **Canvas 코드 배경**으로 와우(WOW) 효과를 만든다.

### 영상 활용
- 소스: AI 생성 영상, Pinterest, Pexels, 실제 촬영
- **중요한 것은 구하는 게 아니라 어떻게 보여줄지**

| 방식 | 특징 | 프롬프트 예시 |
|------|------|---------------|
| 배경 자동 재생 | 히어로 뒤에서 계속 재생, 분위기 연출 | `"히어로 배경에 영상 자동 재생"` |
| 스크롤 연동 | 스크롤 위치에 따라 재생 (애플 스타일) | `"애플 스타일로 스크롤 위치에 따라 영상 재생되게 세팅해줘"` |

### Canvas 코드 배경 (영상이 없을 때)
- `<canvas>` 태그로 실시간 동적 패턴 생성
- 마우스 움직임·스크롤에 반응 가능
- 영상(정해진 것을 재생) vs 캔버스(실시간 생성) → **사용자 행동에 반응 가능**
- 21st.dev에 캔버스 기반 컴포넌트 다수 존재
- 프롬프트: `"캔버스로 인터랙티브한 배경을 만들어줘"`

## 원칙 5 — 와우 모먼트 만들기

사이트에 들어왔을 때 **"와!"** 하게 만드는 장치 하나 — **업종의 본질을 시각화**한다.

### 예시
| 업종 | 본질 | 와우 모먼트 |
|------|------|------------|
| 도배 | 벽에 종이를 입히는 것 | 스크롤하면 도배지가 펼쳐짐 + 마우스 무빙으로 Before/After 시공 비교 |
| 통역 | 언어가 바뀌는 것 | 화면 텍스트가 한국어 → 영어 → 일본어로 전환 |

### AI에게 아이디어 요청하는 법
```
"[업종]의 핵심 가치를 웹에서 코드로 시각화할 수 있는 방법 5가지를 제안해줘.
사용자가 '와!' 하고 느낄 수 있는 인터랙션으로."
```
→ AI가 아이디어를 제안 → 직접 구현 방향 선택

## 5가지 원칙 총정리

| # | 원칙 | 핵심 행동 |
|---|------|-----------|
| 1 | 코딩 전 조사 | 대상·도메인·컴포넌트 3갈래 조사 |
| 2 | 직접 선택 | 눈으로 보고 고르기, DOM 뜯어서 재활용 |
| 3 | 히어로 2라운드 | 테마 먼저 → 색상 확정 |
| 4 | 미디어/캔버스 | 영상 또는 Canvas로 임팩트 추가 |
| 5 | 와우 모먼트 | 업종의 본질을 인터랙션으로 시각화 |

---

## MirAI 프로젝트 적용 포인트

- **원칙 1 (사전 조사)**: 취업 준비생 대상 조사 → 이력서/자소서에서 어떤 언어·강점·키워드가 통하는지 분석 후 UI 설계
- **원칙 3 (히어로 먼저)**: LWW 서비스 등 신규 페이지 작업 시 히어로 섹션을 먼저 2라운드로 확정 → 전체 톤앤매너 통일
- **원칙 5 (와우 모먼트)**: MirAI의 본질 = "AI가 면접관처럼 질문한다" → 입력 텍스트가 질문으로 변환되는 인터랙션, 또는 AI 응답이 타이핑되는 효과 등으로 시각화
