# docs/meetings/

## 목적
팀 회의록 보관. 구두로 논의된 결정사항을 레포로 인코딩하는 곳.

## 구조
```
meetings/
└── YYMMDD_weekly_meeting.md  (날짜별 회의록)
```

- `260214_weekly_meeting.md` — 2/14 멘토 미팅: AI 명세 중요성, 페르소나 면접관 방향, 경쟁사 재정의
- `260221_weekly_meeting.md` — 2/21 멘토 미팅: 워크플로 원칙, 이케아 효과, MVP 극단적 범위 축소
- `260301_weekly_meeting.md` — 3/1 멘토 미팅: SSoT+하네스, 5가지 역량, 프로젝트 목표(한달/10명)

## 역할
- 회의 결정사항 → 레포 SOT로 이동하는 중간 스테이지
- 중요한 결정은 `docs/background/`로, 기능 변경은 `docs/specs/`로 옮겨야 완료
- 파일명 규칙: `YYMMDD_weekly_meeting.md` (예: `260307_weekly_meeting.md`)
