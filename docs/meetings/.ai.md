# docs/meetings/

## 목적
팀 회의록 보관. 구두로 논의된 결정사항을 레포로 인코딩하는 곳.

## 구조
```
meetings/
└── YYYYMMDD_agenda.md  (날짜별 회의록)
```

## 현재 파일
- `20260214_agenda.md` — 2/14 멘토 미팅: AI 명세 중요성, 페르소나 면접관 방향, 경쟁사 재정의
- `20260221_agenda.md` — 2/21 멘토 미팅: 워크플로 원칙, 이케아 효과, MVP 극단적 범위 축소
- `20260301_agenda.md` — 3/1 멘토 미팅: SSoT+하네스, 5가지 역량, 프로젝트 목표(한달/10명)

## 역할
- 회의 결정사항 → 레포 SOT로 이동하는 중간 스테이지
- 중요한 결정은 `docs/background/`로, 기능 변경은 `docs/specs/`로 옮겨야 완료
- 파일명 규칙: `YYYYMMDD_agenda.md` (예: `20260307_agenda.md`)

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
