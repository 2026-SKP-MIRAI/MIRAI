## 1. 타겟 유저 세그먼트

### 1차 타겟 — 취준생

| 항목 | 내용 |
|------|------|
| 연령대 | 20대 초중반 (21~26세) |
| 상황 | 대학 재학·졸업 예정, 공채·수시 준비 중 |
| 기기 친화성 | 모바일 퍼스트. 하루 5~7시간 스마트폰 사용 |
| 주요 채널 | 에브리타임, 오픈카카오, 취업 카페(네이버), 인스타그램, 틱톡 |
| 결제 행동 | 소액 결제 경험 높음 (앱 내 구매, 웹툰 쿠키 등). 큰 금액 일시 지불에 저항 |
| 핵심 니즈 | 저비용·반복 가능한 면접 훈련, 혼자 준비하는 외로움 해소, 진행도 확인 |

### 2차 타겟 — 이직 준비 현직자

| 항목 | 내용 |
|------|------|
| 연령대 | 27~35세 |
| 상황 | 중소·중견기업 재직 2~5년차, 재직 중 이직 준비 |
| 기기 친화성 | 모바일 + PC 병행. 출퇴근 시간 모바일 이용 |
| 핵심 니즈 | 현 직장 노출 없이 조용히 준비, 오래된 면접 감각 회복, 단편 시간 활용 |
| 결제 행동 | 취준생 대비 구매력 높음. 실용적 서비스에 월 구독 지불 의향 있음 |

### 3차 타겟 — 현직자 멘토 (Phase 2 이후)

| 항목 | 내용 |
|------|------|
| 연령대 | 30~40세 |
| 상황 | 대기업·외국계·스타트업 3년차 이상, 면접관 경험 보유 |
| 동기 | 후배 도움 + 플랫폼 내 크레딧 수익 + 자기 브랜딩 |
| 핵심 우려 | 개인 정보 노출, 기업 기밀 유출, 시간 투자 대비 보상 |
| 진입 조건 | 재직 인증(재직증명서 or 회사 이메일), 닉네임 익명 운영 가능 |

## 2. 3가지 페르소나

### 페르소나 1 — 취준생 "이취준"

```
이름: 이취준
나이: 25세
직업: 대학교 4학년 (졸업 예정)
거주지: 서울 관악구 (자취)
```

**상황**
공채 시즌이 다가왔다. 대기업 공채 서류는 통과했지만 면접 준비가 막막하다. 스터디는 구했지만 다들 바빠서 실질적인 도움이 되지 않는다. 유료 컨설팅 광고를 봤지만 1회 30만원이 넘는다. 취준 커뮤니티에서 "혼자 하면 절대 안 된다"는 말을 읽고 불안감이 커졌다.

**목표**
- 대기업 또는 중견기업 최종 합격
- 면접 답변 구조화 역량 향상
- 준비 상황을 객관적으로 점검

**두려움**
- "준비를 충분히 했는지 알 수가 없다"
- "혼자라서 피드백을 받을 수가 없다"
- "돈이 없어서 좋은 컨설팅을 못 받는다"

**하루 루틴**
오전: 자소서 수정 / 오후: 직무 공부 / 저녁: 면접 준비 유튜브 시청 / 취침 전: 스마트폰으로 취업 카페 확인

**JTBD**
> "유료 컨설팅 없이도 체계적으로 면접 준비하고 싶다. 내 답변이 실제로 괜찮은지 즉시 피드백을 받고 싶다."

**Fint 사용 시나리오**
출퇴근 대신 자취방 침대에 누워 스마트폰으로 10분짜리 AI 모의면접을 진행한다. 결과 리포트에서 "두괄식 구조화 점수 62점 → 전날 대비 +8점 상승"을 보고 작은 성취감을 느낀다. 스트릭 7일 달성 배지를 SNS에 공유한다.

### 페르소나 2 — 이직 준비 현직자 "박이직"

```
이름: 박이직
나이: 30세
직업: 중소기업 IT팀 3년차 (재직 중)
거주지: 경기도 성남시
```

**상황**
현 직장에서 3년을 일했다. 연봉 상승 폭이 기대 이하고 성장 환경에 한계를 느낀다. 대기업 또는 외국계 이직을 목표로 하지만, 현 회사에 알려질까 두렵다. 마지막 면접이 3년 전이라 감각이 떨어진 것 같다. 퇴근 후 시간이 없어 학원이나 스터디 참여가 어렵다.

**목표**
- 대기업 or 외국계 이직 성공
- 현 직장 노출 없이 조용히 준비
- 출퇴근 시간 등 틈새 시간 활용

**두려움**
- "현재 회사에 이직 준비 사실이 들킬까"
- "면접 감각이 완전히 떨어졌을까"
- "요즘 면접 트렌드를 모른다"

**하루 루틴**
오전: 지하철 출근 (40분) / 점심: 사내 식사 + 짧은 SNS / 퇴근길: 지하철 (40분) / 야간: 피로로 공부 어려움

**JTBD**
> "직장 다니면서도 틈틈이 면접 감각을 유지하고 싶다. 회사에 들키지 않고, 10분짜리 짧은 세션으로."

**Fint 사용 시나리오**
퇴근 지하철 안에서 이어폰을 꽂고 채팅 형식의 면접 세션을 10분 진행한다. "현직자 페르소나 — 삼성전자 5년차" 페르소나를 선택해 현직자 시각의 피드백을 받는다. 익명이라 현 회사 동료에게 노출될 걱정이 없다.

### 페르소나 3 — 현직자 멘토 "최현직"

```
이름: 최현직
나이: 33세
직업: 대기업 마케팅팀 5년차 (팀장 대행)
거주지: 서울 마포구
```

**상황**
팀에서 신입 면접관으로 3회 참여한 경험이 있다. 취준 시절 멘토가 없어 고생했던 기억이 있어 후배들에게 도움을 주고 싶다. 하지만 회사 이름을 직접 노출하거나 기업 기밀이 외부에 유출되는 것은 절대 안 된다. 보람 있는 활동이면서 약간의 보상도 있으면 좋겠다.

**목표**
- 취준생 후배에게 실질적인 면접 경험 공유
- 재미있고 의미 있는 사이드 활동
- 익명성 보장 하에 활동

**두려움**
- "내 이름·회사가 노출될까"
- "실수로 기업 기밀을 유출하면 어떻게 되나"
- "시간을 투자했는데 아무도 이용하지 않으면 낭비"

**JTBD**
> "익명으로 내 면접 경험을 공유하고, 후배가 합격했을 때 보람을 느끼고 싶다. 크레딧 수익은 보너스."

**Fint 사용 시나리오**
주말 오후, 자신이 실제로 받았던 면접 질문 5개와 답변 전략을 바탕으로 "마케팅 현직자 5년차" 페르소나를 제작한다. 닉네임은 "마케터J", 회사명은 비공개. 일주일 뒤 10명의 취준생이 이 페르소나와 면접을 완료했고 크레딧 N개가 적립됐다. [수량 미확정] 취준생의 감사 쪽지를 익명으로 수신한다.

## 4. 유저 여정 지도

> 취준생 "이취준" 기준 전체 여정

### 여정 개요

```
문제 인식 → 서비스 발견 → 첫 사용(Aha Moment) → 재사용 → 추천
```

---

### 단계별 상세

#### 1단계 — 문제 인식

| 항목 | 내용 |
|------|------|
| **상황** | 대기업 서류 합격 후 면접 준비 방법을 몰라 막막함 |
| **행동** | 취업 커뮤니티 검색, "면접 준비 방법" 유튜브 시청 |
| **감정** | 불안, 초조, 압박감 |
| **페인포인트** | 유료 컨설팅 비용 부담. 믿을 만한 무료 방법 없음 |
| **기회** | "저렴하고 반복 가능한 면접 연습 도구" 수요 명확 |

#### 2단계 — 서비스 발견

| 항목 | 내용 |
|------|------|
| **상황** | 에브리타임 취업 게시판에서 lww 후기 발견. 친구 추천 코드 수신 |
| **행동** | 앱스토어 검색, 스크린샷 확인, 다운로드 |
| **감정** | 호기심, 약간의 회의감 ("또 광고 아닐까") |
| **페인포인트** | "또 하나의 취업 앱"에 대한 피로감 |
| **기회** | 친구 추천·커뮤니티 바이럴이 신뢰 형성에 핵심 |

#### 3단계 — 첫 사용 (Aha Moment)

| 항목 | 내용 |
|------|------|
| **상황** | 직군 설정 → 첫 AI 면접 시작 (로그인 없음 — 선 체험 후 가입. 자소서 업로드는 Phase 1 기능) |
| **행동** | 채팅 UI로 AI 면접관과 10분 대화. 즉시 결과 리포트 수신 |
| **감정** | **"오, 이거 진짜 유용하다"** — 즉각적 가치 체감 |
| **Aha Moment** | 결과 리포트: 답변 구조화 점수, 개선 포인트 3가지, 모범 답변 예시 |
| **페인포인트** | 온보딩 단계가 길면 이탈. 첫 세션까지 3단계 이내로 단축 필수 |

#### 4단계 — 재사용 (Retention)

| 항목 | 내용 |
|------|------|
| **상황** | 매일 스트릭 유지 동기. 크레딧이 쌓이면 현직자 페르소나를 사용하고 싶어짐 |
| **행동** | D1: 다음날 재방문. D7: 스트릭 7일 배지 획득. D30: 레벨 업 |
| **감정** | 성취감, 루틴화, 소속감 (커뮤니티) |
| **페인포인트** | 스트릭이 끊기면 재진입 장벽 상승. 유연한 복구 메커니즘 필요 |
| **기회** | 리텐션 알림: "오늘 스트릭 위기! 5분 세션으로 지키기" |

#### 5단계 — 추천 (Referral)

| 항목 | 내용 |
|------|------|
| **상황** | 면접 합격 후 후기를 쓰고 싶어짐. 친구에게 자연스럽게 공유 |
| **행동** | 결과 리포트 카드 SNS 공유. 추천 코드 발송 |
| **감정** | 자랑, 감사, 보람 |
| **페인포인트** | 공유 콘텐츠가 지나치게 홍보성이면 공유 꺼림 |
| **기회** | 공유 카드는 "내 면접 점수 개선 그래프" 등 개인 성취 중심으로 설계 |

## 5. 실제 유저 페인포인트

> 블라인드·에브리타임·취업 카페 등 커뮤니티 수집 데이터 기반

### 페인포인트 목록

#### P1 — "AI 평가 점수와 실제 합격은 아무 관계 없다"

- **원문 유형**: "AI 면접 90점 받았는데 서류에서 떨어졌어요", "점수 높아도 면접에서 바로 탈락"
- **근본 원인**: AI 피드백이 실제 면접관의 평가 기준과 괴리. 신뢰성 낮음
- **lww 해법**: AI 피드백 + 현직자 피어 피드백 병행. 현직자 "면접관 경험자" 페르소나 활용

#### P2 — "혼자 준비하는 고립감"

- **원문 유형**: "취준하면서 사람을 못 만나서 너무 힘들다", "스터디도 다들 바빠서 형식적으로만 됨"
- **근본 원인**: 취업 준비 특성상 경쟁 구도, 정보 공유 꺼림
- **lww 해법**: 익명 커뮤니티 + 비경쟁적 정보 공유 구조 설계

#### P3 — "반복 연습 비용 누적 부담"

- **원문 유형**: "사람인 AI 면접 10회 하면 10만원이잖아요", "무료 체험 1번만 되고 끝"
- **근본 원인**: 면접 준비는 반복이 핵심인데 반복 비용이 선형 증가
- **lww 해법**: 활동 크레딧 획득 → 무료 반복 사용. 유료 구매는 선택지

#### P4 — "1회성 도구라 결과 받고 끝"

- **원문 유형**: "뭔가 받긴 했는데 그래서 어떻게 해야 하는지 모르겠음", "결과 리포트 보고 끝, 다음 액션 없음"
- **근본 원인**: 개선 방향 없는 점수만 제공
- **lww 해법**: 리포트에 다음 연습 세션 자동 추천. 개선 트래킹 대시보드

#### P5 — "카메라·음성 인식 오류로 연습 흐름 끊김"

- **원문 유형**: "마이크 오류로 5번 다시 시작", "카메라 화질 때문에 평가 신뢰 안 됨"
- **근본 원인**: 화상 면접 도구의 높은 기술 진입 장벽
- **lww 해법**: 텍스트 채팅 우선 UI. 화상은 선택 옵션으로 추후 지원

#### P6 — "개선 방향 불명확"

- **원문 유형**: "두괄식으로 말하세요라고만 하고 어떻게 하라는 건지 예시가 없음"
- **근본 원인**: 추상적 피드백, 구체적 개선 액션 미제시
- **lww 해법**: 피드백 = 문제 진단 + 개선 예시 답변 + 다음 연습 질문 세트

## 6. 취준생 → 현직자 전환 루프

취준생이 합격 후 플랫폼을 떠나지 않고 현직자 멘토로 전환하는 4단계 루프.

```
①합격 후기 작성
    ↓
②현직자 프로필 전환 유도
    ↓
③페르소나 제작 온보딩
    ↓
④첫 페르소나 완료 시 크레딧 보너스
    ↓
(루프) → 페르소나 이용 취준생 증가 → 크레딧 적립 → 멘토 참여 강화
```

### 단계 상세

#### ① 합격 후기 작성
- 트리거: 취준생이 "면접 합격" 상태를 앱에 등록
- 액션: 축하 팝업 + "합격 후기를 작성하고 크레딧 N개를 받으세요 [미확정]" CTA
- 후기 내용: 면접 준비 과정, 실전 질문, 팁 (커뮤니티에 익명 공개 옵션)

#### ② 현직자 프로필 전환 유도
- 트리거: 합격 후기 작성 완료 후
- 액션: "현직자 멘토로 전환하면 크레딧을 벌 수 있어요" 안내 카드
- 조건: 회사 이메일 인증 (즉시, 추천) or 재직증명서 업로드 (3영업일 이내 검토)
- 닉네임 익명 유지, 회사명 비공개 옵션 제공

#### ③ 페르소나 제작 온보딩
- 트리거: 현직자 프로필 전환 완료
- 액션: 페르소나 제작 마법사 (5단계): 직군 선택 → 경력 설정 → 면접 질문 5개 작성 → 답변 전략 설명 → 공개 설정
- 소요 시간: 약 15~20분
- 가이드: AI가 "이런 질문이 취준생에게 인기입니다" 추천

#### ④ 첫 페르소나 완료 시 크레딧 보너스
- 트리거: 첫 페르소나 등록 완료
- 보상: 크레딧 N개 즉시 적립 [수량 미확정 — Phase 1 실데이터 기반 확정 예정]
- 이후: 페르소나 이용 취준생 1명당 크레딧 자동 적립 (현직자 배분율 70%, 수량 미확정)

## 7. 온보딩 플로우

### 취준생 온보딩

> 목표: 첫 방문 후 10분 이내 Aha Moment 도달

```
1. 첫 방문
   └─ 앱 실행, 스플래시 화면 → 온보딩 슬라이드 3장

2. 직군 설정 (로그인 없음 — MVP 선 체험 방식)
   └─ 대분류(IT/마케팅/영업/기획/금융 등) 선택
   └─ 취준 단계 (서류 준비 / 면접 준비 / 최종합격 대기)

3. 자소서 업로드 (선택, Phase 1부터 제공)
   └─ MVP에서는 이 단계 없음 — 기본 질문으로 바로 시작

4. 첫 면접 시작
   └─ "10분짜리 첫 면접을 시작할까요?" CTA
   └─ 채팅 UI로 AI 면접관과 5문항

5. Aha Moment — 결과 리포트 수신
   └─ 답변 구조화 점수 (0~100)
   └─ 잘한 점 2가지
   └─ 개선 포인트 2가지 + 예시 답변
   └─ "저장하려면 로그인" CTA → 자연스러운 가입 유도
   └─ (Phase 1 이후) "내일도 이어서 연습하면 점수가 올라요" → 스트릭 시작

6. 소셜 로그인 (Phase 1 — 결과 저장 동기 후 진입)
   └─ 카카오 / 구글 (1탭)
   └─ 로그인 완료 시 이전 결과 자동 저장
```

**핵심 설계 원칙**:
- 첫 진입 → 첫 면접 완료까지 핵심 입력 1단계 (로그인 없음)
  - 1단계 정의: 직군 + 취준 단계를 1화면에서 동시 입력 → 즉시 면접 시작
  - 자소서 업로드는 Phase 1 선택 사항 (MVP 미포함)
- 자소서 업로드는 선택 (마찰 최소화)
- 결과 리포트는 즉시, 3분 이내 생성

> 목표: 페르소나 첫 등록까지 20분 이내

```
1. 첫 방문
   └─ "취준생으로 가입" vs "현직자로 가입" 선택 화면

   └─ 카카오 / 구글 (1탭)

   └─ "닉네임은 익명으로 유지됩니다" 안내

4. 페르소나 제작
   └─ 직군·경력 설정

   └─ "취준생이 이 페르소나를 이용하면 크레딧이 적립돼요" 안내
   └─ 대시보드: 페르소나 이용 통계, 적립 크레딧 현황
```

**핵심 설계 원칙**:
- 익명성 강조를 온보딩 전 단계에서 반복
- AI 추천으로 페르소나 제작 난이도 낮춤
- 첫 등록 즉시 크레딧 보상으로 초기 동기부여

## 8. AI 에이전트 참여자

### AI·사람 혼재 커뮤니티 개념

Fint는 "사람인지 AI인지 명확히 알 수 없는" 혼재 커뮤니티가 아니다. 오히려 그 혼재를 **투명하게 관리**하는 구조를 설계한다.

#### 핵심 설계 원칙

1. **기본 상태 — 디폴트 오픈**: AI 페르소나는 처음부터 AI임을 표시. 숨기지 않음.
2. **차별점**: 블라인드(모두 사람·익명) vs lww(AI와 사람이 함께 활동하는 것 자체가 정체성)
3. **AI 에이전트 역할**: 현직자가 설정한 페르소나 기반으로 자율 활동. 답변 품질·톤은 현직자가 사전 정의.
4. **투명성**: 이용약관에 "AI와 사람이 함께 활동하는 플랫폼"으로 명시. 개별 게시물에는 AI 배지 표시.

#### 배경 연구 레퍼런스

**Truth Terminal (2024, Andy Ayrey)**
- AI 에이전트가 소셜 미디어 계정을 직접 운영하며 팔로워와 상호작용한 실험적 사례
- lww 함의: AI가 페르소나로서 커뮤니티에 참여할 때 사용자 경험이 사람과 유사할 수 있음. 그러나 신뢰 기반 유지를 위해 투명한 레이어 설계가 필수
- 출처: Ayrey, A. (2024). Truth Terminal. https://truthterminal.com / Wired 커버스토리 (2024년 11월)

**Stanford Generative Agents (2023, Park et al.)**
- 25개의 LLM 기반 AI 에이전트가 소규모 마을 시뮬레이션에서 사회적 상호작용을 하며 사람과 유사한 행동 패턴을 보인 연구
- lww 함의: AI 에이전트가 충분히 풍부한 퍼스낼리티 설정과 메모리 구조를 가질 때 반복적·일관된 면접관 역할이 가능함을 시사
- 출처: Park, J. S., O'Brien, J. C., Cai, C. J., Morris, M. R., Liang, P., & Bernstein, M. S. (2023). Generative Agents: Interactive Simulacra of Human Behavior. *ACM UIST 2023*. https://arxiv.org/abs/2304.03442

#### Fint의 포지션

기존 서비스(블라인드, 커뮤니티 앱)는 AI 봇과 사람이 구분 없이 혼재하여 신뢰 문제를 야기한다. Fint는 이를 역이용한다:

- AI 참여자도 명시적으로 "AI 페르소나"로 등록하고 운영 가이드라인을 준수
- 현직자는 자신의 페르소나 설정(질문 셋, 피드백 기준, 답변 톤)을 정의하고, AI는 이를 충실히 재현
- 취준생은 "AI 면접관도 현직자가 설계한 것"이라는 신뢰를 바탕으로 플랫폼 활용

이 구조는 **AI를 불투명한 위협이 아닌 확장 가능한 멘토링 인프라**로 재정의한다.
