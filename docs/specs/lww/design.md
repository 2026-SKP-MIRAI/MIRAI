# lww 초기 디자인 시안

> 모바일 기준 375px. 브랜드 컬러 #7C3AED(보라)는 사용자가 최종 선택 전 임시 적용 값.
> 이 문서는 개발 레퍼런스용 디자인 명세서다. Figma 확정 전 텍스트 기반 가이드.

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|---|---|
| 친숙함 | 카카오톡 채팅 UI를 레퍼런스로, 학습 없이 즉시 사용 가능 |
| 게임감 | 코인·스트릭·잠금 해제로 지속 참여 동기 부여 |
| 신뢰 | 현직자 기반 콘텐츠임을 시각적으로 강조 |
| 집중 | 각 화면은 하나의 핵심 CTA만 노출 |

---

## 2. 컬러 시스템

> 브랜딩 문서(`branding.md`)에서 최종 컬러 결정 후 이 표를 업데이트한다.

### 기본 팔레트

| 토큰 | 값 (임시) | 용도 |
|---|---|---|
| `color-primary` | `#7C3AED` | 주요 버튼, 유저 채팅 버블, 활성 탭, 강조 요소 |
| `color-primary-light` | `#EDE9FE` | 보라 배경 (카드, 배지, 칩) |
| `color-primary-dark` | `#5B21B6` | 버튼 호버/프레스 상태 |
| `color-coin` | `#F59E0B` | 코인 아이콘, 코인 배지, 코인 관련 CTA |
| `color-coin-light` | `#FEF3C7` | 코인 배경 영역 |
| `color-success` | `#22C55E` | 합격·완료·체크 아이콘 |
| `color-warning` | `#F59E0B` | 경고, 잠금 해제 비용 |
| `color-error` | `#EF4444` | 오류, 코인 부족 알림 |
| `color-bg` | `#FFFFFF` | 페이지 기본 배경 |
| `color-surface` | `#F9FAFB` | 카드, 입력창, AI 채팅 버블 배경 |
| `color-border` | `#E5E7EB` | 구분선, 카드 테두리 |
| `color-text-primary` | `#111827` | 본문 텍스트 |
| `color-text-secondary` | `#6B7280` | 부가 정보, 타임스탬프, 라벨 |
| `color-text-disabled` | `#D1D5DB` | 비활성 텍스트 |
| `color-overlay` | `rgba(0,0,0,0.4)` | 모달 딤 처리 |

### 다크모드 (v2 예정)

v1은 라이트 모드 단일 지원. 다크모드는 v2에서 CSS custom properties 기반으로 활성화.
> **참고**: 다크모드 CSS 토큰은 `branding.md`에 3개 컬러 옵션별로 사전 정의 완료. v1에서는 `.dark` 클래스 미적용, v2에서 그대로 활성화하면 됨.

## 4. 간격 & 그리드

### 기본 단위

4px 기반 스페이싱 시스템.

| 토큰 | 값 | 용도 |
|---|---|---|
| `space-1` | 4px | 아이콘 내부 패딩, 미세 조정 |
| `space-2` | 8px | 아이콘-라벨 간격, 칩 내부 패딩 |
| `space-3` | 12px | 카드 내부 패딩 (소형) |
| `space-4` | 16px | 기본 패딩, 섹션 내부 간격 |
| `space-5` | 20px | 카드 내부 패딩 (대형) |
| `space-6` | 24px | 섹션 간격 |
| `space-8` | 32px | 대형 섹션 구분 |

### 레이아웃

- 좌우 여백(gutter): 16px
- 콘텐츠 최대 너비: 375px (모바일 기준)
- 하단 탭 바 높이: 64px + Safe Area
- 상단 바 높이: 56px
- 상태 바(Status Bar): OS 제공, 컬러 Primary 또는 White

## 8. 인터랙션 & 애니메이션 가이드

### 코인 획득 애니메이션

트리거: 면접 완료, 게시글 작성, 출석 체크

1. 코인 아이콘이 화면 하단에서 상단 코인 배지를 향해 포물선으로 날아간다 (0.6s)
2. 코인 배지 숫자가 증가하며 scale 1.2 → 1.0 bounce (0.3s)
3. 하단에 `+N 🪙` 토스트 표시 (1.5초 후 fade-out)

```
  [🪙 배지]  ← 포물선 이동
      ↑
      |  (arc)
      |
  면접 완료 버튼 주변에서 출발
```

### 스트릭 완료 (7일 달성)

1. 7번째 날 원형이 Primary 컬러로 채워진다 (scale animation 0.4s)
2. 화면 전체에 불꽃 🔥 파티클 2초간 표시
3. 진동 피드백 (haptic: `heavy`) 1회
4. "🎉 7일 연속 달성! 🪙 +N" 모달 표시 [미확정]

### 잠금 해제 애니메이션

1. 코인 CTA 탭 → 코인 차감 확인 모달 표시
2. 확인 탭 → 자물쇠 아이콘이 열리는 애니메이션 (0.4s, rotate + scale)
3. 블러 필터 blur(4px) → blur(0) 전환 (0.3s ease-out)
4. 오버레이 fade-out (0.2s)
5. 콘텐츠 slide-down reveal

### 쪽지 발송

1. "N코인으로 발송" 탭 → 확인 바텀시트 표시
2. 확인 탭 → 코인 차감 애니메이션 (배지 숫자 감소)
3. 발송 중 로딩 인디케이터 (spinner, 1~2초)
4. 성공 시 "💌 쪽지를 보냈어요!" 토스트
5. 실패(네트워크) 시 코인 반환 + 에러 토스트

### 화면 전환

| 전환 유형 | 방향 | 사용 화면 |
|---|---|---|
| Push | 오른쪽 → 왼쪽 slide | 홈 → 상세, 마켓 → 면접 |
| Modal | 아래 → 위 sheet | 코인 확인, 쪽지 발송 |
| Fade | fade-in | 스플래시 → 온보딩 |
| Pop | 왼쪽 → 오른쪽 slide | 뒤로가기 |

### 면접 중 탭 전환 정책 (MVP)

면접 진행 중(`/interview` 채팅 화면 활성 상태)에는 하단 탭 바 탐색을 차단한다.

| 상황 | 동작 |
|---|---|
| 하단 탭 바의 다른 탭 탭 | 탭 이동 없이 현재 화면 유지, 토스트 표시 |
| 시스템 백 버튼 (Android) | "면접을 중단할까요?" 확인 팝업 표시 |
| 스와이프백 (iOS) | "면접을 중단할까요?" 확인 팝업 표시 |

**토스트 메시지**: "면접 중에는 다른 화면으로 이동할 수 없습니다"
- duration: 2000ms
- 스타일: 5-10절 알림(Toast) 기본 스타일 적용

**중단 확인 팝업 (백 버튼 / 스와이프백)**
- 메시지: "면접을 중단할까요? 진행 내용은 저장되지 않습니다"
- 버튼: [취소] (면접 계속) / [중단] (면접 종료 후 이전 화면 이동)
