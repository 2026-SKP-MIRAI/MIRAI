# docs/specs/mirai/

## 목적
MirAI 전체 기능 기술 명세 (`MirAI_proposal.md` v2 기준).

## 구조
```
mirai/
├── dev_spec.md    API 명세·기술 스택·기능별 구현 상세
└── ux_flow.md     UX 흐름·모드 구분·화면별 구성·Open Questions
```

## 역할

| 파일 | 다루는 것 | 다루지 않는 것 |
|------|---------|-------------|
| `dev_spec.md` | API 계약, 기술 스택, 엔진 구조, 프롬프트 지침 | UX 흐름, 화면 구성 |
| `ux_flow.md` | 사용자 여정, 모드 구분, 화면별 UX, Open Questions | API 상세, 기술 구현 |

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
