# MirAI UX 흐름 설계

> **기준 문서:** `MirAI_proposal.md` §5 사용자 경험(UX) 설계 방향
> **최종 업데이트:** 2026-03-05

---

## 1. 전체 사용자 여정

MirAI는 자소서 업로드부터 역량 평가까지 4단계가 끊김 없이 이어진다.

> Week 1 MVP: 인증 없이 기능01만 제공. Week 2 Beta부터 랜딩→인증 단계가 앞에 추가됨.

```
[Week 2+ — 랜딩 / 인증]
  랜딩 → 회원가입/로그인 (Better Auth) → 자소서 업로드로 이동
        ↓

[Step 1 — 서류 분석]
  자소서 PDF 업로드
    ├─ 내 서류에서 나올 맞춤 예상 질문 리스트 확인      (기능 01)
    └─ 면접관 시각의 서류 강점·약점·개선 방향 진단       (기능 02)
          ↓ 생성된 질문을 기반으로 면접 시작

[Step 2 — 면접 시뮬레이션]
  채팅 면접 / AI 아바타 면접 선택                      (기능 06)
  HR·기술팀장·경영진이 함께 참여하는 패널 면접           (기능 03)
  + 답변이 부족할 때 자동 꼬리질문 등장                  (기능 04)
          ↓

[Step 3 — 의지적 연습]
  같은 질문에 "다시 답변하기"로 반복 연습               (기능 05)
          ↓

[Step 4 — 역량 평가]
  8축 역량 점수 + "다음에 무엇을 고쳐야 하는지" 리포트   (기능 07)
```

## 2. 기능 간 데이터 흐름

```
기능01 (자소서 업로드 + 질문 생성)
  └─ 생성된 질문 목록 / resumeId
        ├─→ 기능03 (패널 면접 세션 시작 — questions 또는 resumeId 전달)
        │     └─→ 기능04 (꼬리질문 엔진 — 각 답변마다 내장)
        │           └─→ 기능07 (세션 종료 후 8축 리포트 생성)
        └─→ 기능05 (연습 모드 — 개별 질문 단위 반복 연습)

기능02 (서류 강점·약점 진단)
  └─ 독립 실행 가능 (기능01 이후 또는 단독)
```

> 결정: 기능01 → 기능03 전달 방식은 `resumeId` 기반 서버 재생성. 세션 메모리에 파싱 결과 캐시 후 `resumeId`로 참조.

## 3. 모드 구분 규칙

면접 시작 시 두 가지 모드 중 하나를 선택한다.

| 구분 | 실전 면접 모드 | 연습 면접 모드 |
|------|--------------|--------------|
| 패널 면접 (기능 03) | ✓ | ✓ |
| 꼬리질문 (기능 04) | ✓ | ✓ |
| 즉각 피드백 (기능 05) | **없음** (세션 중 피드백 차단) | ✓ (질문 → 답변 → 피드백 → 재답변) |
| 종료 후 8축 리포트 (기능 07) | ✓ | ✓ |
| AI 아바타 면접 (기능 06) | ✓ 선택 (Phase 4 이후) | ✓ 선택 (Phase 4 이후) |

**실전 모드:** 실제 면접과 동일한 조건. 세션 중 피드백 없음. 종료 후 8축 리포트에서 종합 평가.

**연습 모드:** 의지적 연습 순환 구조 (`질문 → 답변 → 피드백 → 재답변 → 다음 질문`). "다시 답변하기" 버튼으로 재도전 가능.

> API 구현: `/api/interview/start`의 `interviewMode: "real" | "practice"` 파라미터로 제어 (→ `dev_spec.md` 기능03 참조)

## 4. 화면별 구성

### 랜딩 및 인증 흐름 (Week 2 — Better Auth)

```
[랜딩/홈]
  → "시작하기" 클릭
      ├─ 비로그인: 회원가입 / 로그인 화면
      │     → 소셜 로그인 또는 이메일 인증 (Better Auth)
      │     → 완료 후 자소서 업로드 화면(기능01)으로 이동
      └─ 로그인 상태: 자소서 업로드 화면(기능01) 바로 진입
```

**Free tier 제한 UX:**

| 상태 | 동작 |
|------|------|
| 무료 (월 3회 이하) | 기능01 정상 실행 |
| 무료 (월 3회 초과) | "이번 달 무료 횟수를 모두 사용했습니다. Pro로 업그레이드하면 무제한 이용 가능합니다" → 결제 화면 유도 |
| Pro (월 19,900원) | 무제한 실행 |

> MVP(Week 1) 1차 출시에서는 인증·결제 없이 기능01만 제공. 회원가입·결제 흐름은 Week 2 Beta에 추가.

---

### 기능 01 — 자소서 업로드 및 맞춤 예상 질문 생성

- **진입:** 드래그앤드롭 또는 파일 선택으로 PDF 업로드
- **상태 전이:** `idle` → `uploading` → `processing` → `done` / `error`
- **결과 화면:** 카테고리별 예상 질문 리스트

  | 카테고리 | 설명 |
  |---------|------|
  | 직무 역량 | 지원 직무 핵심 역량 관련 |
  | 경험의 구체성 | STAR 구조·수치 근거 확인 |
  | 성과 근거 | 성과의 맥락·임팩트 |
  | 기술 역량 | 기술 스택·구현 방법 |

- **하단 버튼:** "다른 자소서로 다시 하기" (즉시 재시작)

---

### 기능 02 — 서류 강점·약점 진단

- **진입:** 업로드 완료 후 "진단하기" 버튼
- **결과 화면:** 3섹션 구성

  | 섹션 | 내용 |
  |------|------|
  | 강점 | 면접관이 좋게 볼 수 있는 서술 2~4가지 |
  | 약점 | 부족하거나 모호한 부분 2~4가지 |
  | 개선 방향 | 약점 1개당 구체적 개선 가이드 1개 대응 |

---

### 기능 03·04·05 — 면접 시뮬레이션 (채팅 화면)

**모드 선택 → 면접 시작** 순서로 진행.

**채팅 화면 구성:**

```
┌─────────────────────────────────┐
│ [HR 담당자]                      │
│ 팀 프로젝트에서 본인의 역할은?    │
└─────────────────────────────────┘
                ┌──────────────────┐
                │ 저는 백엔드 개발을 │
                │ 담당했고...        │
                └──────────────────┘
┌─────────────────────────────────┐
│ [기술팀장] 🔁 논리 검증           │
│ 구체적으로 어떤 기술을 사용하셨나요? │
└─────────────────────────────────┘
```

- 페르소나 라벨 (HR 담당자 / 기술팀장 / 경영진) 표시
- 꼬리질문 유형 태그 표시: `재확인(CLARIFY)` / `논리 검증(CHALLENGE)` / `심층 탐색(EXPLORE)`
- **연습 모드 전용:** 피드백 표시 후 "다시 답변하기" 버튼

---

### 기능 06 — 채팅 면접 vs AI 아바타 면접 선택 (Phase 4)

| 방식 | 설명 |
|------|------|
| **채팅 면접** | 텍스트 기반. 빠르고 부담 없이 답변 연습 |
| **AI 아바타 면접** | 면접관 아바타가 목소리로 질문. 텍스트 또는 음성 답변 가능 |

**아바타 화면 구성:**
- 채팅 영역 상단에 현재 질문 중인 페르소나 아바타 활성화
- 아바타 상태: `말하는 중` / `대기 중`
- TTS 톤: HR 담당자(친절·부드러움) / 기술팀장(차분·분석적) / 경영진(권위·간결)
- 음성 답변 선택 시: 마이크 버튼 → STT 변환 → 텍스트 확인·수정 후 전송
- TTS 불가 환경: 텍스트 전용으로 대체

> Phase 4 이후 검토 대상: 비언어 분석(시선 처리·말하는 속도·침묵 대처)

---

### 기능 07 — 8축 역량 평가 리포트

- **진입 조건:** 답변 5개 이상 후 세션 종료 (미만 시 "질문을 더 진행해 주세요" 안내)
- **리포트 화면 구성:**
  - 8축 레이더 차트 (한눈에 시각화)
  - 각 역량 아래 실행형 피드백 ("이 질문에 결과 수치를 한 문장 추가하면 좋겠습니다")
  - 성장 곡선 그래프 (세션 누적 시 역량별 점수 변화 — DB 도입 이후 제공 예정)

## 5. Phase별 UX 범위

| Phase | 활성화되는 화면 |
|-------|--------------|
| **MVP** (Week 1) | 기능01 — 자소서 업로드 + 질문 생성 결과 화면 |
| **Phase 1** (Week 2) | + 랜딩 화면, 회원가입/로그인, Free tier 제한 UI, 기능03 채팅 면접, 기능04 꼬리질문 |
| **Phase 2** | + 기능07 8축 리포트 화면 |
| **Phase 3** | + 기능05 연습 모드 피드백·재답변, 기능02 서류 진단 |
| **Phase 4** | + 기능06 AI 아바타 면접 화면 |
