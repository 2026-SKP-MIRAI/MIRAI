# docs/specs/

## 목적
기능별 상세 명세 문서. 기능 구현 전 AC(인수 조건)를 먼저 정의하는 곳.

## 구조
```
specs/
└── (기능별 명세 파일 — Week 1 내 작성 예정)
    예: feature_02_followup_questions.md
        feature_03_persona_simulation.md
        feature_04_competency_evaluation.md
```

## 역할
- TDD의 첫 번째 단계: AC 작성 = 첫 번째 테스트
- PR 리뷰 시 구현이 AC를 충족하는지 확인하는 기준
- 기능 2~4 상세 기획 완성 마감: Week 1 (3/7)
