# docs/specs/

## 목적
기획/기술 설계/배경 문서. "어떻게 만드나"를 다루는 곳.
이슈별 AC(인수 조건)는 GitHub Issue body에 작성한다 — 여기가 아님.

## 구조
```
specs/
├── MVP_dev_spec.md                   MVP 기술 명세 원본 (v1 보존, MVP_proposal.md 기준)
├── MirAI_dev_spec.md                 MVP 기술 명세 최신판 (MirAI_proposal.md 기준)
└── (기능별 설계 문서 — 기능이 복잡할 때만 작성)
    예: spec_followup_questions.md
        spec_persona_simulation.md
        spec_competency_evaluation.md
```

## 역할
- 기능을 구현하기 전 "무엇을 어떻게 만드나" 설계 기준
- API 명세, 시스템 구성도, 기술 스택 결정 등
- PR 리뷰 시 구현이 설계와 일치하는지 확인하는 기준
- 이슈별 AC 체크리스트 → GitHub Issue body
- 의사결정 근거 → docs/decisions/ (ADR)

## 규칙 변경 시 컨펌 필요
이 파일에 기재된 규칙과 충돌하는 변경을 하려는 경우, 작업을 진행하기 전에 반드시 사람(팀원)에게 먼저 확인을 받아야 한다.
