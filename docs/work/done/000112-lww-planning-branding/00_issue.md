## 서비스 기획 필수 구성 요소

> 신규 서비스 기획 시 반드시 다뤄야 할 항목. 순서보다 누락 방지가 목적.

### 1. 문제 정의
- **페인포인트**: 타겟 유저가 겪는 핵심 문제 (3개 이내, 감성 맥락 포함)
- **가치 제안**: 이 서비스가 왜 존재하는가 — 한 문장으로

### 2. 시장
- **TAM** (전체 시장): 이 문제를 가진 사람이 전 세계/국내 몇 명인가
- **SAM** (서비스 가능 시장): 우리가 실제 도달 가능한 세그먼트
- **SOM** (점유 가능 시장): MVP 기준 현실적으로 먹을 수 있는 크기
- 시장 성장 트렌드 (성장 중인지, 정체인지)

### 3. 타겟 유저
- **페르소나**: 나이·직업·상황·목표·두려움
- **JTBD** (Jobs to be Done): 유저가 이 서비스를 "고용"하는 이유
- **유저 여정**: 문제 인식 → 해결 시도 → 서비스 발견 → 첫 사용 → 재사용
- **유저 전환 루프**: 취준생 → 합격 → 현직자 → 이직 준비 → 재취준 사이클 설계 ★
- **AI 에이전트 참여자**: AI 페르소나도 플랫폼 참여자 — 사람과 AI가 뒤섞인 커뮤니티 ★

### 4. 경쟁사 분석
- **직접 경쟁사**: 같은 문제를 푸는 서비스
- **간접 경쟁사**: 유저가 지금 쓰는 대안 (엑셀, 유튜브 등)
- **포지셔닝 매트릭스**: 2축으로 경쟁사 배치 → 공백 포지션 확인
- **차별점**: 경쟁사 약점 기반으로 도출 (추상적 슬로건 X)
- **커뮤니티 구조 차별화**: 블라인드(모두 사람·익명) vs lww(AI·사람 혼재, 🤖 배지 기본 공개 — 디폴트 오픈) ★

### 5. 수익 모델
- **수익원**: 직접(구독·거래 수수료·광고) + 간접(데이터·B2B)
- **멀티사이드 플랫폼이면**: 각 사이드별 인센티브 구조 (누가 왜 머무나)
- **콜드 스타트 해결책**: 첫 번째 사이드 진입 전략
- **단가 × 볼륨**: 수익 구조의 현실성 검증
- **코인/현금 레이어 구분**: 외부 검증된 수익쉐어 모델이면 현금, 그 외엔 코인 ★
- **가상화폐(코인) 법적 성격**: 영구 게임머니(현금화 불가) vs 선불업 등록 필요 여부 명시 ★

### 6. 성장 전략
- **Phase 구분**: MVP → 확장 → 수익화 (각 Phase 목표·지표 명시)
- **퍼널**: 유입 → 활성화 → 리텐션 → 수익화 → 추천 (AARRR)
- **그로스 레버**: 바이럴 계수, 리텐션 루프, 네트워크 효과
- **유저 전환 루프**: 취업 성공 취준생 → 현직자 전환 온보딩 플로우 ★
- **AI 에이전트 생태계 비전**: Phase별 AI 자율 활동 → AI 외주 경제 (Truth Terminal·Virtuals Protocol 레퍼런스) ★
- **MVP 범위 원칙** ★ : MVP = 로그인 없음 + 코인 없음 + Aha Moment까지만. 인증·재화·소셜 기능은 Aha Moment 검증 후 Phase 1에서 추가. MVP가 복잡할수록 첫 가치 검증이 늦어짐.
- **Phase 명명 원칙** ★ : MVP / Phase 1 / Phase 1 후반 / Phase 2+3 (4단계). "Phase 3" 단독 표기 금지 — Phase 2+3 병합 고정.

### 7. 핵심 지표 (KPI)
- **North Star Metric**: 서비스 건강을 나타내는 단 하나의 지표
- **선행 지표**: NSM에 영향을 주는 액션 지표
- **목표 수치**: Phase별 구체적 숫자 (MAU, 전환율, ARPU 등)

### 8. 브랜딩
- **핵심 슬로건**: 서비스를 한 문장으로 (lww: "취업준비부터 직장생활까지, 재밌게" 확정) ★
- **키 컬러**: 경쟁사 컬러 회피 후 선정, HEX 코드 + 선정 근거
- **컬러 팔레트**: Primary / Secondary / Neutral / Semantic
- **타이포그래피**: 폰트 + 사이즈 스케일
- **톤앤매너**: 브랜드 키워드 3개, 카피라이팅 가이드

### 9. 온보딩 플로우
- 첫 방문 → 가입 → 핵심 기능 첫 경험 → 습관 형성까지 단계별 설계
- 각 단계 이탈 원인과 해결책
- Aha Moment 특정 (유저가 "이거다" 느끼는 순간)
- **유저 타입별 온보딩 분리**: 취준생 vs 현직자 온보딩 플로우 별도 설계 ★
- **선 체험 후 가입 패턴** ★ : 가입 없이 핵심 기능을 먼저 체험하게 한 뒤, Aha Moment(결과 리포트) 직후 "저장하려면 로그인" CTA로 자연 가입 유도. 로그인 게이트 없이 즉시 체험 진입 → 전환율 극대화
  - ❌ 기각 패턴: 결과 리포트 자체를 로그인 뒤에 공개 (Aha Moment 전 게이트 = 이탈)
  - ✅ 채택 패턴: 결과 리포트 공개 → 저장 유도 CTA → 소셜 로그인 (Phase 1)

### 10. IA · 화면 설계
- **IA**: 서비스 전체 정보 구조 (메뉴·화면 계층)
- **핵심 화면 플로우**: 주요 태스크별 화면 이동 경로
- **스토리보드**: 핵심 화면 목업 (모바일 기준)
- **빈 상태 / 에러 상태**: 데이터 없을 때, 실패했을 때 화면 정의
