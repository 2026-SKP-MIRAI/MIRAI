# chore: dev_spec.md 기능 05 연습모드 단순화 및 Supabase DB 도입 내용 반영

## 목적
`docs/specs/mirai/dev_spec.md` 스펙 문서를 실제 구현 방향에 맞게 수정한다.

## 배경
1. **기능 05 연습모드**: 가이드 모드·비교 모드는 실제 구현 방향과 다름. 사용자가 답변하면 즉각 피드백을 주고, 재답변 한 번 후 그 답변에 대한 피드백을 제공하는 단일 흐름(자유 모드)만 운영한다.
2. **Supabase DB 도입**: 기능 03·04 구현 이후 각 서비스에 Supabase DB를 연동함.
3. **기능 07 `growthCurve`**: 아키텍처 불변식("엔진은 stateless, 데이터 저장은 서비스 책임")에 따라 **엔진은 항상 `null`을 반환**한다. `growthCurve`는 서비스(Next.js)가 Supabase에서 과거 세션 이력을 조회하여 조립한 뒤 클라이언트에 제공한다. 엔진 응답 스펙의 `null`은 유지하되, 주석만 이 구조를 명확히 반영하도록 수정한다.
