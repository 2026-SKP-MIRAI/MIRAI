# feat: [engine] 프롬프트 엔지니어링 고도화 — HR·기술팀장·경영진 페르소나 v2

## 사용자 관점 목표
면접관 페르소나별로 실제 현업 면접관처럼 깊이 있고 특색 있는 질문을 받을 수 있다.

## 배경
현재 HR·기술팀장·경영진 세 페르소나 프롬프트는 최소 기능 구현 수준의 프롬프팅으로 작성되어 있다.
모든 기능 구현이 완료된 이후, 각 페르소나의 특성을 살린 고도화된 프롬프트로 교체하는 develop 작업이 필요하다.

> ⚠️ **작업 시점**: 모든 기능(이슈) 구현 완료 후 진행. 기능 개발 중에는 건드리지 않는다.

### 공통 문제점

| 문제 | 설명 |
|------|------|
| **역할 정의 피상적** | "당신은 채용 HR 담당자입니다" 한 줄로 페르소나 깊이 없음 |
| **평가 기준 부재** | 무엇을 기준으로 질문해야 하는지 명시 없음 → LLM이 일반적·뻔한 질문 생성 |
| **질문 전략 없음** | 질문 개수·난이도·순서·유형 분배 가이드 없음 |
| **이력서 분석 지시 없음** | 이력서에서 무엇을 추출해야 하는지 안내 없음 → 피상적 독해 |
| **Few-shot 예시 없음** | 기대 출력 수준을 보여주는 예시가 전혀 없음 |
| **꼬리질문 페르소나 미분화** | 3개 페르소나 모두 동일한 꼬리질문 프롬프트 사용 |

## 적용할 프롬프트 엔지니어링 기법 (10가지)

> 근거: LLM 프롬프팅 기법 논문 5편 분석 (ReAct, Tree of Thoughts, Self-Reflection, Graph of Thoughts, Step-Back Prompting) + 추가 기법 조사

### 논문 기반 기법 (5종)

#### A. Step-Back Prompting (추상화 → 추론)
- **출처**: Step-Back Prompting 논문 (Google DeepMind)
- **원리**: 구체적 세부사항에서 한 걸음 물러나 고수준 개념/원리를 먼저 도출 → 그 위에서 추론
- **적용**: 이력서 읽기 전에 "이 직무에서 핵심적으로 검증해야 할 역량은 무엇인가?" 먼저 정의
  - HR: "조직 적합성을 판단하는 근본 원리는?" → 그 원리 기반 질문 설계
  - 기술팀장: "이 기술 스택에서 실무 역량의 본질은?" → 깊이 있는 기술 질문
  - 경영진: "비즈니스 임팩트를 만드는 리더의 핵심 자질은?" → 전략적 질문
- **근거**: TimeQA에서 +27% 향상, Hard 난이도 문제 42.6%→62.3% (논문 실험)

#### B. Chain-of-Thought (CoT) — 단계적 추론
- **출처**: Wei et al. (2022) + ReAct/ToT 논문 내 비교 분석
- **원리**: 최종 출력 전에 중간 추론 단계를 명시적으로 거치게 함
- **적용**: `<analysis>` 섹션에서 이력서 핵심 포인트 3~5개 추출 → 각 포인트에 대해 질문 설계
- **근거**: ReAct 논문에서 CoT 적용 시 환각 14%→6% 감소

#### C. Self-Reflection (자기 성찰·검증)
- **출처**: Renze & Guven, Johns Hopkins University
- **원리**: LLM이 생성한 결과를 스스로 검토·교정
- **적용**: 프롬프트 말미에 자기 검증 체크리스트 삽입
  - "생성한 질문이 이력서에 실제 기재된 내용에 근거하는가?"
  - "뻔한 일반 질문(자기소개, 지원동기)이 아닌 이력서 맞춤 질문인가?"
  - "페르소나 고유의 관점이 반영되었는가?"
  - "질문 난이도가 적절히 분포되었는가?"
- **근거**: GPT-4에서 +23.5% 정확도 향상, 모든 LLM에서 유의미 (p < 0.001)

#### D. ReAct 패턴 (추론 ↔ 행동 교차)
- **출처**: Yao et al. (2022)
- **원리**: Thought → Action → Observation을 교차 배치하여 단계별 목표 추적
- **적용**: 꼬리질문 프롬프트에서 활용
  1. Thought: "지원자의 답변에서 검증이 필요한 부분은 X이다"
  2. Action: "CHALLENGE 유형으로 X에 대한 근거를 요구하는 꼬리질문 생성"
  3. Observation: "이 질문이 새로운 정보를 이끌어낼 수 있는가 판단"
- **근거**: 환각 기반 질문 감소, ALFWorld에서 33-90% 성능 향상

#### E. Graph of Thoughts 원리 (분해 → 병합)
- **출처**: GoT 논문 (2023-2024)
- **원리**: 복잡한 작업을 하위 작업으로 분해 → 독립 처리 → 집계
- **적용**: 기술팀장 프롬프트에서 이력서를 3영역으로 독립 분석 후 통합
  - 영역 A: 기술 스택 깊이 (원리 이해도)
  - 영역 B: 설계/아키텍처 역량 (트레이드오프)
  - 영역 C: 문제 해결 패턴 (디버깅·장애 대응)
- **근거**: ToT 대비 62% 오류 감소, 비용 31% 절감

### 추가 기법 (5종)

#### F. Few-Shot Exemplars (소수 예시 학습)
- **적용**: 각 페르소나에 좋은 질문 vs 나쁜 질문 예시 2~3쌍 제공
- 예시: `[나쁜] "팀 프로젝트 경험이 있나요?"` → `[좋은] "이력서의 A 프로젝트에서 팀원 간 기술 의견 충돌 시 본인은 어떤 역할로 조율했고 결과는?"`

#### G. Persona Depth Anchoring (페르소나 깊이 고정)
- **적용**: 역할을 직함이 아닌 구체적 경험·관점·판단 기준으로 정의
  - HR: "15년 경력 채용 전문가. STAR 기법 기반 행동면접 전문. 문화 핏 미스매치로 인한 조기 퇴사를 줄이는 것이 최우선 목표"
  - 기술팀장: "10년+ 시니어 엔지니어. 왜 이 설계를 선택했는가를 항상 묻는 사람"
  - 경영진: "CTO/VP 레벨. 이 사람이 6개월 후 어떤 임팩트를 만들 수 있는가로 판단"

#### H. Structured Output Scaffolding (구조화된 출력 스캐폴딩)
- **적용**: 질문 유형·난이도 분포를 명시적으로 강제
  - 기술팀장 예: "기술 깊이(Why/How) 40%, 시스템 설계 30%, 문제 해결 시나리오 20%, 기술 커뮤니케이션 10%"
  - 난이도: "기본 확인 20%, 심화 탐색 50%, 도전 질문 30%"

#### I. Negative Constraint Prompting (금지 조건 명시)
- **적용**: Yes/No 닫힌 질문 금지, 이력서에 없는 내용 질문 금지, 자기소개·지원동기 같은 공통 질문 금지, 질문 간 중복 금지

#### J. Role-Conditional Reasoning (역할 조건부 추론)
- **적용**: 동일한 이력서 포인트에 대해 페르소나마다 다른 관점으로 질문 유도
  - 같은 "프로젝트 경험"에 대해: HR→협업 관점, 기술팀장→기술 관점, 경영진→사업 관점

## 페르소나별 v2 프롬프트 구조 설계

### HR v2 (`interview_hr_v2.md`)
**적용 기법**: Step-Back + CoT + Few-Shot + Self-Reflection + Negative Constraints

```
1. [Persona Depth] 15년 경력 HR, STAR 기법 전문, 문화 핏 전문가
2. [Step-Back] "이 지원자의 조직 적합성을 판단하기 위해 검증할 핵심 소프트스킬 영역 3~5개 도출"
3. [CoT] <analysis> 블록: 이력서에서 행동 패턴 단서 추출 → STAR 기반 질문 설계
4. [Few-Shot] 좋은/나쁜 질문 예시 2~3쌍 (행동 기반, 상황 판단, 가치관)
5. [Output Scaffold] 행동 기반(STAR) 40%, 상황 판단 30%, 문화 핏 20%, 동기/성장 10%
6. [Self-Reflection] 이력서 근거 여부·열린 질문 여부·HR 관점 반영 여부 자기 검증
7. [Negative Constraints] 닫힌 질문·일반 질문·기술 심화 질문 금지
```

**핵심 질문 패턴**:
- **STAR 행동 질문**: "이력서의 X 경험에서 구체적으로 어떤 상황(S)이었고, 본인의 역할(T), 행동(A), 결과(R)는?"
- **문화 핏 시나리오**: "팀의 방향성과 본인의 의견이 다를 때 어떻게 대처하시나요?"
- **동기 탐색**: "이력서에서 Z 분야로의 전환이 보이는데, 이 변화의 배경은?"

### 기술팀장 v2 (`interview_tech_lead_v2.md`)
**적용 기법**: Step-Back + CoT + GoT 분해 + Few-Shot + Self-Reflection

```
1. [Persona Depth] 10년+ 시니어 엔지니어, 코드 리뷰 전문가
2. [Step-Back] "이 기술 스택/도메인에서 실무 역량의 본질적 구성 요소는?"
3. [GoT 분해] 영역 A(기술 깊이) / 영역 B(설계/아키텍처) / 영역 C(문제 해결) 독립 분석
4. [CoT] 각 영역별 추론 → 질문 도출
5. [Few-Shot] 표면적 질문 vs 깊이 있는 질문 대비 2~3쌍
6. [Output Scaffold] 기술 깊이 40%, 시스템 설계 30%, 문제 해결 20%, 기술 커뮤니케이션 10%
7. [Self-Reflection] 유행어 나열 질문 아닌가? 사고 과정을 드러내는 질문인가?
8. [Negative Constraints] 단답형 지식 확인·HR 영역 질문 금지
```

**핵심 질문 패턴**:
- **기술 깊이**: "이력서에 X 기술을 사용했다고 되어 있는데, 왜 대안(Y, Z) 대신 X를 선택했나요? 트레이드오프는?"
- **시스템 설계**: "이력서의 A 프로젝트를 10배 트래픽으로 확장한다면 아키텍처를 어떻게 변경?"
- **문제 해결**: "프로덕션에서 B 시스템 응답 지연이 갑자기 3배로 늘었습니다. 어디서부터 디버깅?"

### 경영진 v2 (`interview_executive_v2.md`)
**적용 기법**: Step-Back + CoT + Role-Conditional + Few-Shot + Self-Reflection

```
1. [Persona Depth] CTO/VP 레벨, ROI·스케일러빌리티 중심, "6개월 후 어떤 변화?"
2. [Step-Back] "비즈니스 임팩트를 만드는 인재의 핵심 자질은?"
3. [CoT] 이력서에서 임팩트 단서 추출 → 경영진 관점 검증 질문 도출
4. [Role-Conditional] 프로젝트 경험→"비즈니스 가치 환산하면?" / 기술 경험→"조직 전체에 어떤 영향?"
5. [Few-Shot] 경영진 질문 예시 2~3쌍
6. [Output Scaffold] 비즈니스 임팩트 30%, 리더십 30%, 전략적 사고 25%, 비전/성장 15%
7. [Self-Reflection + Negative Constraints] 기술 세부·단순 인성 질문 금지
```

**핵심 질문 패턴**:
- **비즈니스 임팩트**: "이력서의 X 프로젝트가 조직 매출이나 효율에 구체적으로 어떤 기여? 수치로?"
- **리더십**: "Y팀을 이끈 경험에서 팀원의 성장에 본인이 기여한 구체적 사례는?"
- **전략적 사고**: "현재 산업의 가장 큰 변화 3가지는 무엇이고, 입사 후 어떻게 대응?"

### 꼬리질문 v2 (`interview_followup_v2.md`)
**적용 기법**: ReAct + Persona-Conditional + Self-Reflection + Few-Shot

```
1. [Persona-Conditional] 페르소나별 스타일 분화
   - HR: 감정·동기·가치관 탐색. 부드럽지만 깊이 있게.
   - 기술팀장: 기술적 근거·대안·트레이드오프 추궁. 코드/설계 수준으로.
   - 경영진: 비즈니스 임팩트·확장성 검증. "So what?" 관점.

2. [ReAct] Thought→Action→Observation 교차
   - Thought: "답변에서 [구체성 부족/논리 비약/흥미로운 단서] 발견"
   - Action: "[CLARIFY/CHALLENGE/EXPLORE] 유형 꼬리질문 생성"
   - Observation: "새로운 정보를 이끌어낼 수 있는가?"

3. [CLARIFY/CHALLENGE/EXPLORE 페르소나별 세분화]
   CLARIFY:
   - HR: "좀 더 구체적인 상황을 말씀해주실 수 있나요?"
   - 기술팀장: "구체적으로 어떤 코드/설계로 구현했나요?"
   - 경영진: "그 결과를 숫자로 말씀해주시겠어요?"

   CHALLENGE:
   - HR: "반대 상황이었다면 어떻게 대처했을까요?"
   - 기술팀장: "그 접근의 단점은 없었나요? 대안은 고려했나요?"
   - 경영진: "경쟁사도 같은 전략을 쓴다면 차별점은?"

   EXPLORE:
   - HR: "그 경험이 본인의 커리어 방향에 어떤 영향을 주었나요?"
   - 기술팀장: "그 기술을 다른 문제에도 적용해본 경험이 있나요?"
   - 경영진: "그 경험을 더 큰 규모에 적용한다면?"

4. [Self-Reflection] 기답변 반복 아닌가? 면접 깊이를 더하는가? 페르소나 관점 반영?
5. [Few-Shot] 페르소나별 꼬리질문 예시 1~2개씩
6. [Negative Constraints] 불필요한 추가 질문·주제 이탈·압박 면접 톤 금지
```

### K. 구조화 면접 패턴 도입 (참고 자료 기반 핵심 기법)

에이치닷 질문지의 **문항 구조**를 v2 프롬프트의 질문 생성 패턴으로 채택한다:

**1) 경험면접 + 상황면접 2종 분리**
- **경험면접**: 과거 실제 경험을 묻는 질문 (STAR 패턴)
  - 예: "다른 사람들과 일을 함께할 때, 난관에 부딪혀 많은 사람들이 힘들어하고 포기하려고 했던 경험에 대하여 자신은 어떻게 행동하였는지를 중심으로 말씀해주시기 바랍니다."
- **상황면접**: 가상 시나리오를 제시하고 대처 방법을 묻는 질문
  - 예: "당신은 인사팀 팀장입니다. 그 동안 주기적으로 실시하였던 인사평가 자료를 토대로 연말 연봉인상과 인센티브를 적용하려 합니다. 인사평가에 적절한 자료는 무엇이 있으며, 그것을 어떻게 활용하여 평가할 것인지 말씀해 주시기 바랍니다."
- **v2 적용**: HR은 경험면접 중심(70%), 기술팀장은 상황면접 비중 높임(50%), 경영진은 혼합(50:50)

**2) 추가 질문(꼬리질문) 표준 패턴 6개**
에이치닷 질문지의 추가 질문은 아래 6가지 패턴으로 구조화되어 있으며, 꼬리질문 v2의 Few-Shot 예시로 직접 활용:
1. 상황/배경을 구체적으로 설명해달라
2. 그렇게 행동한 이유는?
3. 그때 어떤 생각이 들었는가?
4. 다른 사람들의 반응은?
5. 결과는 어떠했는가?
6. 이 경험을 통해 배운 점은?

**3) 질문 의도 명시 → Self-Reflection 체크리스트로 활용**
각 질문에는 "이 질문으로 검증하려는 것"이 3~5개 명시되어 있다:
- 예 (긍정성 Q1): ① 다른 사람들에게 긍정적인 영향력을 미쳤는가? ② 긍정적인 태도로 부정적인 정서를 회복하였는가? ③ 어려움을 극복하면서 새롭게 얻은 깨달음이 있는가?
- **v2 적용**: 프롬프트에 "각 질문을 생성할 때 해당 질문의 검증 의도를 내부적으로 먼저 정의하라"는 CoT 지시 삽입

**4) 답변 체크포인트 (Positive/Negative) → 꼬리질문 판단 기준**
각 문항에 Positive (+1) 항목 5개, Negative (-1) 항목 5개가 정의되어 있다:
- Positive 예: "힘든 상황에서 좌절하지 않고 긍정적인 사고와 행동을 하였다"
- Negative 예: "어려움이 닥쳤을 때 좌절하고 포기하려 하였다"
- **v2 적용**: 꼬리질문 프롬프트에서 답변 판단 시 "이 답변이 Positive 체크포인트에 해당하는가, Negative에 해당하는가"를 기준으로 CLARIFY/CHALLENGE/EXPLORE 유형 결정

### 30개 역량 체계 → 페르소나별 매핑

에이치닷의 4대 역량군 30개 역량을 MirAI 3개 페르소나에 매핑:

| 역량군 | 세부 역량 | HR | 기술팀장 | 경영진 |
|--------|---------|:---:|:-------:|:------:|
| **성과역량** (12) | 긍정성, 성취갈망, 열정, 기획력, 정보관리력, 문제해결력, 성실성, 끈기, 실행력, 창의력, 성과관리력, 전문분야 지식/기술 | 긍정성, 성실성, 끈기 | 문제해결력, 기획력, 정보관리력, 실행력, 전문분야 지식/기술, 창의력 | 성취갈망, 열정, 성과관리력 |
| **관계역량** (7) | 고객지향성, 설득력, 의사소통력, 능동성, 유연성, 협력성, 협상력 | 의사소통력, 협력성, 유연성, 능동성 | 의사소통력 (기술 커뮤니케이션) | 설득력, 협상력, 고객지향성 |
| **적응역량** (5) | 규범성, 책임감, 모범성, 스트레스 내성, 스트레스 복원력 | 규범성, 책임감, 스트레스 복원력 | 책임감 | 스트레스 내성 |
| **리더십역량** (6) | 통합조정력, 비전제시력, 변화촉진력, 업무위임력, 의사결정력, 조직계발력 | - | 의사결정력 | 비전제시력, 통합조정력, 변화촉진력, 업무위임력, 조직계발력 |

### 수습평가 체계 → 평가 관점 보강

수습평가 문항의 3대 분류를 면접 질문의 숨은 평가 관점으로 활용:

| 분류 | 세부 역량 | 면접 질문 관점 |
|------|---------|-------------|
| **성과** | 긍정성(환경긍정/자기긍정), 적극성(성취추구/성장추구), 성실성(유지행동/점검행동), 전략성(인지력/분석력) | "이 사람이 입사 후 성과를 낼 수 있는가?" |
| **관계** | 사회성(맥락이해/관계대응), 정체성(수용성/능동성), 관계성(우호성/협력성) | "이 사람이 팀에 잘 녹아들 수 있는가?" |
| **적응** | 객관성(도덕성/책임성), 비활성(부정성/태만성 - 역채점), 과활성(불안성/충동성 - 역채점) | "이 사람이 조직에 적응하고 장기근속할 수 있는가?" |

### 면접 가이드 → Negative Constraints 강화

MIDAS 면접 가이드에서 추출한 금지사항을 v2 프롬프트에 직접 반영:

**법적 금지 질문** (채용절차법 제4조 3항):
- 개인/가족사, 성별, 나이, 단점, 치부 부각 금지
- 정치 성향, 신체적 차별, 직무 무관 민감한 개인사 금지
- 부모 직업, 가족 관계, 출신 학교 평가 금지

**면접 품질 금지사항**:
- "야근이 많은 직무인데 괜찮은가요?" 류 압박 질문 금지
- "스펙이 다른 분들에 비해 안 좋네요" 류 비하 발언 금지
- 부정적 피드백, 훈계 톤 금지

**좋은 질문 패턴** (가이드에서 검증된 것):
- "불가능하거나 힘든 상황이었음에도 불구하고 무엇인가를 시도했던 경험을 말씀해주세요" (긍정성)
- "본인이 수긍할 수 없는 상황에서 무엇인가를 진행해야 했던 경험에 대해 말씀해주세요" (유연성)
- "자신의 가치와 조직의 가치가 다를 경우 어떻게 대처하실 건가요?" (문화 핏)
- "합격 통보 후 면접을 어떻게 준비했는지 말씀해주세요" (성실성/전략성)

### Few-Shot 예시 소스 정리

v2 프롬프트에 삽입할 Few-Shot 예시는 아래 자료에서 직접 발췌·각색:

| 페르소나 | Few-Shot 소스 | 예시 유형 |
|---------|-------------|---------|

| **기술팀장** | 에이치닷 문제해결력·전문분야 지식/기술·실행력 문항 (상황면접 위주) | 상황면접 + 기술 트레이드오프 질문 |
| **경영진** | 에이치닷 비전제시력·통합조정력·성과관리력 문항 | 경험면접 + 비즈니스 임팩트 질문 |
