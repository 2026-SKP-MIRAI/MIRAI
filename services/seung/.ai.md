# services/seung/ — 이승현

## 목적
이승현의 개인 서비스. 자소서 기반 모의면접 시스템을 end-to-end로 구현.
DDD (Domain-Driven Design) + TDD 적용.

## 역할
- 기능 흐름: 자소서 업로드 → 질문 생성 → 패널 면접 시뮬레이션 → (향후) 역량 평가
- engine/ 만 호출 — LLM·파서 직접 구현 금지
- Week 1 MVP 완료: 자소서 → 질문 생성 end-to-end 작동
- Week 2~3: 패널 면접 + 꼬리질문 구현 (Phase 1 완료)
- Week 4: 특색 기능 1개 이상 + 배포 URL 제출

## 인증·DB·스토리지
- **인증**: Supabase Auth (Better Auth 미사용)
  - 클라이언트: `@supabase/ssr` createBrowserClient + `NEXT_PUBLIC_SUPABASE_URL`/`NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - 서버: createServerClient (서버 컴포넌트·API Route에서 사용)
- **DB**: Supabase PostgreSQL
  - 런타임: `DATABASE_URL` (pooler, port 6543, `?pgbouncer=true`)
  - 마이그레이션: `DIRECT_URL` (Session Pooler, port 5432 — IPv4 환경에서 Direct 연결 불가)
  - Prisma로 테이블 생성 시 RLS 자동 활성화 안됨 — 별도 SQL 마이그레이션 필수
- **파일 저장**: Supabase Storage (AWS S3 미사용)
  - 버킷명: `SUPABASE_STORAGE_BUCKET` 환경변수 사용 (하드코딩 금지)
  - 서버 경유 업로드 (WAF 보호 유지)
- **보안**: `SUPABASE_SERVICE_ROLE_KEY`는 서버 전용 — `NEXT_PUBLIC_` 접두사 절대 금지
- 환경변수 상세: `docs/specs/mirai/dev_spec.md` §6

## 배포
- **방식**: Docker + ECR + GitHub Actions (`deploy-seung.yml`)
- **URL**: `seung.mirainterview.com` (ALB → Route53 → HTTPS)
- **포트**: 3000
- **헬스체크**: `GET /api/health`
- **런타임 환경변수**: EC2 `~/.env.seung`
- **빌드 타임 환경변수**: GitHub Secrets (`SEUNG_NEXT_PUBLIC_SUPABASE_URL`, `SEUNG_NEXT_PUBLIC_SUPABASE_ANON_KEY`)

- `ENGINE_BASE_URL` — FastAPI 엔진 주소 (같은 VPC 내 private IP 사용)
- `DATABASE_URL` — Supabase PostgreSQL pooler (port 6543, `?pgbouncer=true`)
- `DIRECT_URL` — Supabase PostgreSQL direct (port 5432, 마이그레이션 전용)
- `SUPABASE_URL` — Supabase 프로젝트 URL (서버 전용)
- `SUPABASE_ANON_KEY` — Supabase 익명 키 (서버 전용)
- `SUPABASE_SERVICE_ROLE_KEY` — Supabase 서비스 롤 키 (서버 전용, RLS 우회)
- `SUPABASE_STORAGE_BUCKET` — Supabase Storage 버킷명
- `NEXT_PUBLIC_SUPABASE_URL` — Supabase URL (빌드 타임 주입)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — Supabase 익명 키 (빌드 타임 주입)
- `E2E_AUTH_BYPASS` — **프로덕션 배포 시 절대 설정 금지**. `1`로 설정하면 `__e2e_bypass=1` 쿠키만으로 미들웨어 인증을 우회할 수 있음. CI E2E 테스트 전용 (`playwright.config.ts`에서만 사용)
