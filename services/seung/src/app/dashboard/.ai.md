# dashboard

> 이슈 #157 — 내 면접 기록 대시보드

## 목적

로그인한 사용자의 자소서 업로드 기록, 면접 세션 수, 역량 리포트, 서류 진단 결과를 한눈에 조회하고 이어볼 수 있는 대시보드.

로그인 후 기본 진입 경로(`/`) 및 로그인 완료 후 리다이렉트 목적지.

## 구조

```
dashboard/
├── page.tsx   # Client Component — /api/dashboard 호출, 카드 목록 렌더링
└── .ai.md     # 이 파일
```

## 역할

- `GET /api/dashboard` 호출 → `DashboardResumeItem[]` 수신
- 자소서별 카드 렌더링: PDF 파일명, 업로드 날짜, 면접 세션 수, 리포트 링크, 서류 진단 링크
- 빈 상태(empty state): 자소서 없을 때 "새 면접 시작" CTA
- 헤더 "새 면접 시작" 버튼 → `/resume` (새 업로드)
- 카드 "이 자소서로 다시 면접하기" 버튼 → `/resume?resumeId={id}` (업로드 스킵, 바로 모드 선택)
- 카드 "삭제" 버튼 → `DELETE /api/resume/[id]` → Report + Session + Resume cascade 삭제

## 데이터 흐름

```
DashboardPage (Client Component)
  → fetch('/api/dashboard')
  → GET /api/dashboard (route.ts)
  → prisma.resume.findMany({ where: { userId }, include: sessions + report })
  → DashboardResponse { resumes: DashboardResumeItem[] }

ResumeCard "삭제" 버튼
  → window.confirm()
  → DELETE /api/resume/[id] (route.ts)
  → prisma.$transaction([deleteMany(report), deleteMany(session), delete(resume)])
  → onDelete(id) → 로컬 상태에서 제거
```
