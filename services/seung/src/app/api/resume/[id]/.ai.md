# api/resume/[id]

> 이슈 #157 — 자소서 삭제 API

## 목적

특정 자소서와 연관된 면접 세션, 리포트를 cascade 삭제하는 API 엔드포인트.

## 구조

```
api/resume/[id]/
├── route.ts   # DELETE /api/resume/[id] 핸들러
└── .ai.md     # 이 파일
```

## 역할

- `DELETE /api/resume/[id]`
  - Supabase Auth로 userId 확인 → 없으면 401
  - `prisma.resume.findUnique({ where: { id } })` → 없으면 404
  - `resume.userId !== user.id` → 403 (소유권 검증)
  - `prisma.$transaction([deleteMany(report), deleteMany(session), delete(resume)])` — cascade 삭제
    - Prisma 스키마에 `onDelete: Cascade` 없으므로 수동 트랜잭션
    - 삭제 순서: Report → InterviewSession → Resume (FK 의존 순서)
  - 성공 시 204 No Content
