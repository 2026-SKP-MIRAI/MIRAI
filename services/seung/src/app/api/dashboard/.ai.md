# api/dashboard

> 이슈 #157 — 내 면접 기록 대시보드

## 목적

로그인한 사용자의 자소서 목록과 각 자소서별 면접 통계를 반환하는 API 엔드포인트.

## 구조

```
api/dashboard/
├── route.ts   # GET /api/dashboard 핸들러
└── .ai.md     # 이 파일
```

## 역할

- `GET /api/dashboard`
  - Supabase Auth로 userId 확인 → 없으면 401
  - `prisma.resume.findMany({ where: { userId }, include: sessions + report })` 조회
  - 응답: `{ resumes: DashboardResumeItem[] }` (최신순)
  - 각 항목: `id`, `createdAt`, `fileName` (null이면 날짜 폴백), `sessionCount`, `hasReport`, `reportId`, `hasDiagnosis`

## 관련 파일

- `src/app/dashboard/page.tsx` — 이 API를 호출하는 Client Component
- `src/lib/types.ts` — `DashboardResumeItem`, `DashboardResponse` 타입
- `tests/api/dashboard.test.ts` — Vitest 단위 테스트 (4케이스)
