# services/kwan/ — 김관우

## 목적
김관우의 개인 서비스. 자소서 기반 모의면접 시스템을 end-to-end로 구현.
DDD (Domain-Driven Design) + TDD 적용.

## 현재 진행 상태
- **Week 1 MVP 완료** (2026-03-09): 자소서 PDF 업로드 → 질문 생성 end-to-end 작동
- **Week 2 기능 완료** (2026-03-11): 패널 면접 세션 + 꼬리질문 연동 (engine #40 기반)
- **Phase 3 완료** (2026-03-20): analyze 전환 + PDF Supabase 저장 + 기능 02·05·07 구현

## 기술 스택
- **Next.js 16** (App Router, RSC)
- **Prisma 6** + PostgreSQL (Supabase) — DB ORM
- **Vitest** — 단위 테스트 (API routes: node 환경, components: jsdom 환경)
- **@supabase/supabase-js** — PDF 스토리지 (Service Role Key 사용)
- **Playwright** — e2e 테스트

## API 라우트

### `POST /api/resume/questions`
- PDF 수신 → buffer 추출 → engine `/api/resume/analyze`로 resumeText + targetRole 추출
- `Promise.all([engine /api/resume/questions({ resumeText, targetRole }), Prisma resumes INSERT])` 병렬
- 반환: `{ resumeId, questions, meta }`
- PDF fire-and-forget 업로드: `uploads/${uuid}.pdf` → storageKey DB 업데이트

### `POST /api/resume/feedback`
- 입력: `{ resumeId, targetRole? }`
- targetRole 폴백: 제공값 → inferredTargetRole → '미지정 직무'
- engine `POST /api/resume/feedback { resumeText, targetRole }` 호출
- diagnosisResult DB 저장 (실패해도 결과 반환)
- 반환: `{ scores (5축), strengths, weaknesses, suggestions }`

### `GET /api/resume/diagnosis`
- 입력: `?resumeId=`
- Prisma resume.diagnosisResult 반환
- diagnosisResult 없으면 404

### `POST /api/practice/feedback`
- 입력: `{ question, answer, previousAnswer? }`
- stateless — DB 없음
- engine `POST /api/practice/feedback` 호출
- 반환: `{ score, feedback, keywords, improvedAnswerGuide, comparisonDelta? }`

### `POST /api/report/generate`
- 입력: `{ sessionId }`
- sessionComplete 확인, history.length >= 5 확인
- 멱등성: 기존 리포트 있으면 200 반환, 없으면 engine 호출 후 DB 저장 → 201
- P2002 동시 요청 시 findUnique fallback → 200
- 반환: `{ reportId }`

### `GET /api/report`
- 입력: `?reportId=`
- 반환: `{ id, sessionId, totalScore, scores (8축), summary, axisFeedbacks, createdAt }`

### `POST /api/interview/start`
- 입력: `{ resumeId, mode? }` — mode: 'practice' | 'real' (기본 'real')
- engine `POST /api/interview/start { resumeText, personas, mode: 'panel' }` 호출
- session.interviewMode 저장

### `POST /api/interview/answer`
- 입력: `{ sessionId, answer }`

### `GET /api/interview/session`
- 입력: `?sessionId=`
- 반환에 `interviewMode` 포함

## DB 스키마 (Prisma)

### Resume
| 필드 | 타입 | 설명 |
|------|------|------|
| id | String (cuid) | PK |
| resumeText | String | 추출된 이력서 텍스트 |
| questions | Json | 생성된 질문 목록 |
| storageKey | String? | Supabase 스토리지 경로 |
| inferredTargetRole | String? | analyze 응답에서 추출된 직무 |
| diagnosisResult | Json? | 5축 진단 결과 |
| createdAt | DateTime | |

| 필드 | 타입 | 설명 |
|------|------|------|
| interviewMode | String | 'real' \| 'practice' (기본 'real') |
| report | Report? | 1:1 리포트 관계 |
| (기존 필드) | ... | |

| 필드 | 타입 | 설명 |

## 역할
- 기능 흐름: 자소서 업로드 → 질문 생성 → 진단 (기능 02) → 면접 시뮬레이션 → 리포트 (기능 07)
- engine/ 만 호출 — LLM 직접 구현 금지 (HTTP: ENGINE_BASE_URL)
- **Week 1 (완료)**: MVP — 자소서 → 질문 생성 end-to-end
- **Week 2 (완료)**: 패널 면접 세션 + 꼬리질문 연동
- **Phase 3 (완료)**: analyze 전환 + PDF 저장 + 기능 02·05·07 + 연습 모드 UI (모드 선택 + 피드백 패널 + 재답변)
- 테스트: 14개 파일, 112개 TC 전체 PASS

## 불변식
```
1. import anthropic, import fitz 금지 — 엔진만 LLM 사용
2. ENGINE_BASE_URL 환경변수로 엔진 통신
3. 에러 메시지는 한국어로 사용자 노출
4. 테스트 없는 PR 금지 (npm test 통과 필수)
5. DATABASE_URL / DIRECT_URL — NEXT_PUBLIC_ 접두사 금지
6. Supabase 스토리지 클라이언트는 Service Role Key 사용 (인증 없음 서비스)
```
