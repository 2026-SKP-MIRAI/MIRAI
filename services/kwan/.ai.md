# services/kwan/ — 김관우

## 목적
김관우의 개인 서비스. 자소서 기반 모의면접 시스템을 end-to-end로 구현.
DDD (Domain-Driven Design) + TDD 적용.

## 현재 진행 상태
- **Week 1 MVP 완료** (2026-03-09): 자소서 PDF 업로드 → 질문 생성 end-to-end 작동
- **Week 2 기능 완료** (2026-03-11): 패널 면접 세션 + 꼬리질문 연동 (engine #40 기반)

## 기술 스택
- **Next.js 16** (App Router, RSC)
- **Prisma 6** + PostgreSQL (Supabase) — DB ORM
- **pdf-parse 2** — 서버 사이드 PDF 텍스트 추출 (면접 세션 DB 저장용)
- **Vitest** — 단위 테스트 (API routes: node 환경, components: jsdom 환경)
- **Playwright** — e2e 테스트

## API 라우트

### `POST /api/resume/questions`
- PDF 수신 → pdf-parse로 resumeText 추출 → Prisma `resumes` INSERT → resumeId 획득
- 동일 PDF를 engine `/api/resume/questions` 포워딩 → questions 수신
- resumes 테이블에 questions UPDATE
- 반환: `{ resumeId, questions, meta }`
- **참고**: 서비스에서 PDF 텍스트를 직접 추출하는 이유 — 엔진 `/api/resume/questions` 응답에 추출 텍스트가 없고, 면접 세션(`/api/interview/start`)에서 `resumeText`가 필요하기 때문. LLM 호출 없이 텍스트 추출만 수행하므로 아키텍처 불변식(LLM은 엔진에서만) 위반 아님.

### `POST /api/interview/start`
- 입력: `{ resumeId }`
- Prisma에서 resumeText 조회
- engine `POST /api/interview/start { resumeText, personas, mode: 'panel' }` 호출
- Prisma `interview_sessions` 생성 (currentPersonaLabel, currentQuestionType 포함)
- 반환: `{ sessionId, firstQuestion }`

### `POST /api/interview/answer`
- 입력: `{ sessionId, answer }` (answer는 trim 후 최대 5000자)
- Prisma에서 session + resume 조회
- engine `POST /api/interview/answer` 호출 (history, questionsQueue, currentQuestion, currentPersona 전달)
- Prisma session 업데이트 (history에 questionType 포함)
- 반환: `{ nextQuestion, sessionComplete }`

### `GET /api/interview/session`
- 입력: `?sessionId=`
- Prisma session 조회 → history, currentQuestion, sessionComplete 등 반환
- 페이지 새로고침 시 면접 상태 복원 목적

## 환경변수 (`.env.local` 필요 — `.env.local.example` 참고)
```
ENGINE_BASE_URL=http://localhost:8000
DATABASE_URL=postgresql://[user]:[pw]@[host]:6543/postgres?pgbouncer=true&sslmode=require
DIRECT_URL=postgresql://[user]:[pw]@[host]:5432/postgres?sslmode=require
```
주의: `NEXT_PUBLIC_` 접두사 금지 (서버 전용)

Supabase 콘솔에서 RLS 정책 수동 추가 필요.

## 역할
- 기능 흐름: 자소서 업로드 → 질문 생성 → 면접 시뮬레이션 → (Week 3) 역량 평가
- engine/ 만 호출 — LLM 직접 구현 금지 (HTTP: ENGINE_BASE_URL)
- **Week 1 (완료)**: MVP — 자소서 → 질문 생성 end-to-end
- **Week 2 (완료)**: 패널 면접 세션 + 꼬리질문 연동 + URL 라우팅 + 세션 복원
- Week 3~4: 역량 평가 + 특색 기능 + 배포

## 불변식
```
1. import anthropic, import fitz 금지 — 엔진만 LLM 사용
2. ENGINE_BASE_URL 환경변수로 엔진 통신 (타임아웃 30초)
3. 에러 메시지는 한국어로 사용자 노출
4. 테스트 없는 PR 금지 (npm test 통과 필수)
5. DATABASE_URL / DIRECT_URL — NEXT_PUBLIC_ 접두사 금지
```
