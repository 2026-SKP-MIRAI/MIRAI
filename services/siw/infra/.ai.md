# services/siw/infra/ — AWS 인프라 자동화

## 목적
Airflow EC2 인스턴스의 비용 절감을 위한 스케줄 자동 on/off 인프라.
DAG 실행 시간(UTC 15:00)에만 EC2를 켜고, 나머지 시간은 꺼둔다.

## 구조
```
infra/
├── lambda/
│   ├── ec2_start/
│   │   ├── handler.py          # EC2 StartInstances Lambda (boto3)
│   │   └── requirements.txt    # 빈 파일 (boto3 Lambda 런타임 내장)
│   ├── ec2_stop/
│   │   ├── handler.py          # EC2 StopInstances Lambda (boto3)
│   │   └── requirements.txt
│   └── tests/
│       ├── conftest.py         # moto mock_aws fixture + EC2 가상 인스턴스 생성
│       ├── test_ec2_start.py   # ec2-start 단위 테스트 (pytest + moto)
│       └── test_ec2_stop.py    # ec2-stop 단위 테스트 (pytest + moto)
├── iam-policy.json             # Lambda 실행 역할 최소 권한 정책 문서
├── deploy.sh                   # AWS CLI 배포 스크립트 (멱등)
└── .ai.md                      # 이 파일
```

## 스케줄
| 규칙 | cron (UTC) | KST | 동작 |
|------|-----------|-----|------|
| airflow-ec2-start | `cron(45 14 * * ? *)` | 23:45 | EC2 시작 |
| airflow-ec2-stop | `cron(0 16 * * ? *)` | 01:00 | EC2 중지 |

## 배포 방법
```bash
export EC2_INSTANCE_ID=i-0123456789abcdef0
export LAMBDA_ROLE_ARN=arn:aws:iam::123456789012:role/lambda-ec2-scheduler
./infra/deploy.sh
```

사전 조건:
- AWS CLI 설치 + `aws configure` 완료
- Lambda 실행 역할(IAM Role) 사전 생성 — `iam-policy.json` 정책 부여 필요

## 테스트
```bash
cd infra/lambda
pip install moto[ec2] pytest boto3
pytest tests/ -v
```

## 환경변수 (Lambda)
| 변수 | 설명 |
|------|------|
| `EC2_INSTANCE_ID` | 타겟 EC2 인스턴스 ID |
| `AWS_REGION` | 리전 (기본값: ap-northeast-2) |
