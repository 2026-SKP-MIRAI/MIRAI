# airflow/ — LLM 품질 모니터링 배치

## 목적
S3에 적재된 LLM 이벤트 로그를 일별로 집계하여 analytics DB에 저장한다.

## 구조
- dags/llm_quality_dag.py: 일별 집계 DAG (extract → aggregate → load → alert)
- sql/001_create_llm_events_daily.sql: analytics DB DDL

## 실행 조건 (Phase B)
1. EC2 배포 완료 + S3 버킷 mirai-logs 생성
2. Airflow Variable 설정: S3_LOG_BUCKET, S3_LOG_PREFIX, ANALYTICS_DB_CONN
3. Airflow Connection 설정: analytics_db (PostgreSQL)
