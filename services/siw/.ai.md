## 역할
- 기능 흐름: 자소서 업로드 → 질문 생성 → 면접 시뮬레이션 → (Week 3~) 역량 평가
- engine/ 만 호출 — LLM·파서 직접 구현 금지
- ENGINE_BASE_URL 환경변수로 엔진 접근 (기본값: http://localhost:8000)
- siw가 Prisma/Supabase로 면접 세션 상태 관리 (engine은 stateless)
- Week 1: MVP 완료 (자소서 → 질문 생성 end-to-end 작동)
- Phase 1: 패널 면접 세션 + 꼬리질문 엔진 연동 완료
- Week 4: 특색 기능 1개 이상 + 배포 URL 제출

## 기술 스택
- Next.js 15, React 19, TypeScript (strict)
- Vitest 2, @testing-library/react 16, Playwright (e2e)
- Prisma 7 + @prisma/client + **@prisma/adapter-pg** (v7 드라이버 어댑터 필수)
- **pdf-parse v2** (`PDFParse` 클래스 API, `src/lib/pdf-parser.ts` 추상화)
- `serverExternalPackages: ["pdf-parse"]` — next.config.ts에서 webpack 번들링 제외
- @supabase/ssr, @supabase/supabase-js

## 인증·DB·스토리지
- **인증**: Supabase Auth (Better Auth 미사용)
  - 클라이언트: `@supabase/ssr` createBrowserClient + `NEXT_PUBLIC_SUPABASE_URL`/`NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - 서버: createServerClient + `middleware.ts` (`getUser()`로 세션 갱신 필수)
- **DB**: Supabase PostgreSQL + Prisma ORM
  - 런타임: `DATABASE_URL` (pooler, port 6543, `?pgbouncer=true`)
  - 마이그레이션: `DIRECT_URL` (direct, port 5432)
  - Prisma로 테이블 생성 시 RLS 자동 활성화 안됨 — 별도 SQL 마이그레이션 필수
  - InterviewSession 모델: id (cuid), resumeText, currentQuestion, currentPersona, questionsQueue (Json), history (Json), sessionComplete, createdAt
- **파일 저장**: Supabase Storage (AWS S3 미사용)
  - 버킷명: `SUPABASE_STORAGE_BUCKET` 환경변수 사용 (하드코딩 금지)
  - 서버 경유 업로드 (WAF 보호 유지)
- **보안**: `SUPABASE_SERVICE_ROLE_KEY`는 서버 전용 — `NEXT_PUBLIC_` 접두사 절대 금지
- 환경변수 상세: `docs/specs/mirai/dev_spec.md` §6

## 아키텍처 불변식
- engine은 stateless — 인증·DB 없음
- siw가 Prisma/Supabase로 면접 세션 상태 관리
- engine 수정 없이 siw에서 세션 영속성 구현
- RLS: Supabase에서 interview_sessions 테이블에 별도 SQL로 활성화 필요
