description: 이슈를 중도 포기한다. 사유를 입력받고 이슈를 닫은 뒤 워크트리·브랜치를 정리한다. 사용법: /drop-issue [이슈번호|짧은이름]
---

## 인수 형식 (모두 선택사항)

`$ARGUMENTS`는 다음 형식 중 하나거나 생략 가능하다:

| 형식 | 예시 | 동작 |
|------|------|------|
| 생략 | (없음) | 현재 브랜치에서 자동 감지 |
| 이슈번호만 | `15` | 이슈번호로 브랜치·워크트리 탐색 |
| 6자리 패딩+짧은이름 | `000015-my-feature` | 직접 브랜치명 구성 |
| 짧은이름만 | `my-feature` | 브랜치 목록에서 매칭 |

## 실행 순서

### 1. 브랜치·이슈번호 확정

**인수 없는 경우 (워크트리 내 실행):**
```
git branch --show-current
```
결과 예: `feat/000015-pdf-upload` → type=`feat`, PADDED=`000015`, 짧은이름=`pdf-upload`, 이슈번호=`15`

**이슈번호(정수)만 있는 경우:**
```
git branch | grep {zero-padded 번호}
```

**`000015-my-feature` 형식:**
```
git branch | grep 000015-my-feature
```

**짧은이름만 있는 경우:**
```
git branch | grep {짧은이름}
```

매칭된 브랜치가 없으면 오류 안내 후 중단.
- BRANCH = 확정된 브랜치명
- 이슈번호 = PADDED에서 정수 변환 (앞의 0 제거)
- WORKTREE = `.worktree/{PADDED}-{짧은이름}`
- WORKFOLDER = `docs/work/active/{PADDED}-{짧은이름}`

### 2. CWD 감지

현재 작업 디렉토리가 대상 워크트리 내부인지 확인한다:
```
pwd
```

현재 경로가 `{WORKTREE}/` 하위이면 실행을 중단하고 안내한다:
```
⚠️  현재 대상 워크트리 내부에서 실행 중입니다.
메인 워크트리로 이동 후 다시 실행하세요:

  cd {메인 워크트리 경로}
  /drop-issue {이슈번호}
```

### 3. 미커밋 변경사항 확인

워크트리에 미저장 변경사항이 있는지 확인한다:
```
git -C {WORKTREE} status --short
```

출력이 비어 있지 않으면 경고를 출력하고 사용자 확인을 받는다:
```
⚠️  미저장 변경사항이 있습니다. 포기 시 모두 삭제됩니다.

{변경 파일 목록}

계속하시겠습니까? (y/n)
```

`n` 선택 시 중단한다.

### 4. 포기 사유 입력

포기 사유를 입력받는다 (빈 값 허용 안 함):
```
포기 사유를 입력하세요 (필수):
```

빈 값이면 다시 요청한다.

### 5. GitHub 이슈 코멘트

포기 사유를 이슈에 코멘트로 남긴다:
```
gh issue comment {이슈번호} --body "## 이슈 포기

**사유:** {입력된 사유}

**날짜:** {오늘 날짜}"
```

### 6. 이슈 Close

이슈를 닫는다:
```
gh issue close {이슈번호} --reason "not planned"
```

### 7. 작업 폴더 이동

`{WORKFOLDER}`가 존재하면:

1. `{WORKFOLDER}/00_issue.md` 하단에 포기 정보를 추가한다:
```
---

## 포기 기록

- **날짜**: {오늘 날짜}
- **사유**: {입력된 사유}
```

2. 폴더를 done으로 이동한다 (`DROPPED-` 접두사로 완료 작업과 구분):
```
mv {WORKFOLDER} docs/work/done/DROPPED-{PADDED}-{짧은이름}
```

존재하지 않으면 이 단계를 건너뛴다.

### 8. Worktree 삭제

```
git worktree remove --force {WORKTREE}
```

실패하거나 존재하지 않으면:
```
이미 삭제됐거나 존재하지 않는 워크트리입니다. 브랜치 정리를 계속합니다.
```

### 9. 로컬 브랜치 삭제

```
git branch | grep {PADDED}-{짧은이름}
```

찾은 브랜치를 강제 삭제한다:
```
git branch -D {브랜치명}
```

없으면:
```
이미 삭제됐거나 존재하지 않는 브랜치입니다.
```

### 10. 리모트 브랜치 삭제

리모트 브랜치 존재 여부를 확인한다:
```
git ls-remote --heads origin {브랜치명}
```

존재하면 삭제한다:
```
git push origin --delete {브랜치명}
```

존재하지 않으면 이 단계를 건너뛴다.

### 11. 완료 안내

```
✓ 이슈 #{이슈번호} 포기 처리 완료

- 이슈 상태: CLOSED (not planned)
- 작업 폴더: docs/work/done/DROPPED-{PADDED}-{짧은이름}
- 워크트리: 삭제됨
- 브랜치: 삭제됨
```
