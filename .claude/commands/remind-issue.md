## 인수 형식 (모두 선택사항)

`$ARGUMENTS`는 다음 형식 중 하나거나 생략 가능하다:

| 형식 | 예시 | 동작 |
|------|------|------|
| 생략 | (없음) | 현재 브랜치에서 자동 감지 |
| 이슈번호만 | `15` | 이슈번호로 브랜치 탐색 |
| 6자리 패딩+짧은이름 | `000015-my-feature` | 직접 경로 구성 |
| 짧은이름만 | `my-feature` | 브랜치 목록에서 매칭 |

## 실행 순서

### 1. 브랜치·이슈번호 확정

**인수 없는 경우:**
```
git branch --show-current
```
결과 예: `feat/000015-pdf-upload` → PADDED=`000015`, 짧은이름=`pdf-upload`, 이슈번호=`15`

**이슈번호(정수)만 있는 경우:**
```
git branch | grep {zero-padded 번호}
```

**`000015-my-feature` 또는 짧은이름 형식:**
```
git branch | grep {패턴}
```

매칭 없으면 오류 안내 후 중단.
- WORKFOLDER = `docs/work/active/{PADDED}-{짧은이름}`

### 2. AC 목록 추출

`{WORKFOLDER}/00_issue.md`가 존재하면 읽어서 `## 완료 기준` 또는 `## AC` 섹션의 항목을 추출한다.

파일이 없으면:
```
gh issue view {이슈번호} --json body --jq '.body'
```
로 이슈 body에서 AC 섹션을 추출한다.

AC가 없으면 AC 섹션 없이 변경 파일·다음 작업만 출력한다.

### 3. AC 충족 여부 추정

브랜치 전체 변경사항을 수집한다 (대상 워크트리 내부 또는 CWD 기준):
```
git diff main...HEAD   # 커밋된 변경사항
git diff --cached      # 스테이징된 변경사항
git diff               # 미스테이징 변경사항
```

인수로 이슈번호·짧은이름을 지정한 경우 대상 워크트리 경로를 명시한다:
```
git -C {WORKTREE} diff main...HEAD
```

각 AC 항목에 대해 변경된 파일·코드를 기준으로 충족 여부를 추정한다.

> **주의**: 판단은 추정이며, 동작 테스트나 실행 결과가 아닌 코드 구조 기준입니다. 일부 AC는 직접 확인이 필요할 수 있습니다.

### 4. 현황 출력

다음 형식으로 출력한다:

```
📋 이슈 #{이슈번호} 현황 — {이슈 제목}

## 완료 기준 (추정)
- [x] {AC 항목 1} — {근거 또는 확인 방법}
- [ ] {AC 항목 2} — 미완료
- [ ] {AC 항목 3} — 미완료 (현재 작업 중)

## 브랜치 변경 파일 (main 기준)
  A  {신규 파일}
  M  {수정 파일}

## 다음 작업
- {미완료 AC 기반 다음 할 일}
- {미완료 AC 기반 다음 할 일}
```

### 5. 작업 내역 기록

`{WORKFOLDER}/00_issue.md`가 존재하면 `## 작업 내역` 섹션에 현황 스냅샷을 추가한다.

오늘 날짜 항목이 이미 있으면 **해당 항목을 업데이트**한다 (skip 아님).

추가 형식:
```
### {오늘 날짜}

**현황**: {완료된 AC 수}/{전체 AC 수} 완료

**완료된 항목**:
- {완료 AC 목록}

**미완료 항목**:
- {미완료 AC 목록}

**변경 파일**: {변경된 파일 수}개
```
