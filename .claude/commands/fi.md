## 인수 형식 (모두 선택사항)

`$ARGUMENTS`는 다음 형식 중 하나거나 생략 가능하다:

| 형식 | 예시 | 동작 |
|------|------|------|
| 생략 | (없음) | 현재 브랜치에서 자동 감지 |
| 이슈번호만 | `15` | 이슈번호로 브랜치 탐색 |
| 6자리 패딩+짧은이름 | `000015-my-feature` | 직접 브랜치명 구성 |
| 짧은이름만 | `my-feature` | 브랜치 목록에서 매칭 |

## 실행 순서

### 1. 브랜치·이슈번호 확정

**인수 없는 경우 (워크트리 내 실행):**
```
git branch --show-current
```
결과 예: `feat/000015-pdf-upload` → type=`feat`, PADDED=`000015`, 짧은이름=`pdf-upload`, 이슈번호=`15`

**이슈번호(정수)만 있는 경우:**
```
git branch | grep {zero-padded 번호}
```
매칭된 브랜치명에서 type·짧은이름 추출.

**`000015-my-feature` 형식:**
```
git branch | grep 000015-my-feature
```

**짧은이름만 있는 경우:**
```
git branch | grep {짧은이름}
```

매칭된 브랜치가 없으면 오류 안내 후 중단.
- BRANCH = 확정된 브랜치명
- 이슈번호 = PADDED에서 정수 변환 (앞의 0 제거)

### 2. 변경사항 확인

다음 명령으로 변경사항을 파악한다:
```
git status
git diff --stat
```

### 3. 커밋 메시지 자동 생성

`git diff HEAD` 또는 `git diff`로 변경 내용을 분석한다.
변경된 파일과 내용을 바탕으로 커밋 메시지를 생성한다.

형식: `{type}: {변경 내용 요약} (#{이슈번호})`

예시:
- `feat: PDF 업로드 엔드포인트 구현 (#15)`
- `fix: 파서 예외 처리 추가 (#23)`
- `docs: API 인터페이스 명세 업데이트 (#31)`

### 4. 코드 리뷰

커밋 전, `git diff HEAD` 또는 `git diff`로 변경사항 전체를 검토한다.
다음 항목을 중심으로 확인한다:

- **버그·로직 오류**: 잘못된 조건, 경계값, 예외 처리 누락
- **불변식 위반**: `CLAUDE.md` 기재 불변식 (LLM 호출 위치, PDF 파싱 위치 등)
- **테스트 누락**: 새 코드에 대응하는 테스트가 없는 경우
- **코드 품질**: 중복 코드, 불명확한 변수명, 불필요한 복잡성

발견된 문제를 아래 형식으로 출력한다:
```
🔍 코드 리뷰 결과

[ISSUE] {파일명}:{라인번호} — {문제 설명}
[ISSUE] ...

문제를 수정하고 싶으신가요? (수정 후 다시 /finish-issue를 실행하거나, 이대로 진행할 수 있습니다)
```

발견된 문제가 없으면:
```
✓ 코드 리뷰 통과 — 특이사항 없음
```

사용자가 수정하겠다고 하면 작업을 중단한다. 이대로 진행하겠다고 하면 다음 단계로 진행한다.

### 5. .ai.md 업데이트 확인

작업 대상 디렉토리의 `.ai.md`가 최신 상태인지 확인한다.

```
⚠️  .ai.md 업데이트 확인
작업한 디렉토리의 .ai.md 에 변경사항이 반영됐나요?
- 새로 추가된 파일·모듈 기술
- 구조 변경 사항 반영
- 완료된 항목 업데이트

업데이트가 필요하면 먼저 수정 후 진행하세요.
```

사용자 확인 후 다음 단계로 진행한다.

### 6. 사용자 확인

생성한 커밋 메시지를 보여주고 확인을 받는다:
```
커밋 메시지: "{생성된 메시지}"
이대로 진행할까요? (수정이 필요하면 알려주세요)
```

확인 후 다음을 실행한다:
```
git add .
git commit -m "{커밋 메시지}"
```

변경사항이 이미 커밋된 경우 커밋 단계를 건너뛰고 push로 진행한다.

### 7. Push

다음 명령을 실행한다:
```
git push origin {BRANCH}
```

### 8. PR 생성

다음 명령을 실행한다:
```
gh pr create --title "{커밋 메시지}" --body "Closes #{이슈번호}"
```

### 9. 완료 안내

다음 형식으로 출력한다:
```
✓ PR 생성 완료: {PR URL}

리뷰어로 멘토가 자동 지정됩니다.
PR 머지 후 /cleanup-issue {이슈번호} 을 실행하세요.
```
